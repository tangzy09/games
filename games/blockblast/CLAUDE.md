# CLAUDE.md — games/blockblast

**Cube Blast: Block Puzzle**（内部代号 `blockblast`）：8×8 方块消除拼图。托盘给 3 块不可旋转的拼图，拖上棋盘，填满整行/整列即消除；三块全放完才补新三块；**剩余的每一块都放不下 = 结束**。root `CLAUDE.md`（引擎契约、部署铁律、git 纪律、iOS 流水线）先读，本文件只讲本游戏专属。

**规格是权威**：`DESIGN.md`（改核心前必查；数值全部由 `tools/sim.js` 跑模拟校准，不是拍脑袋）。

## 这个游戏的唯一卖点：预生成块流（别动它）

`dealer.js` 的全部公开面就一个纯函数：

```js
stream(seed, i) -> piece      // 第 i 块是什么，完全由 (seed, i) 决定
```

**签名里没有 board、没有 score** —— 想作弊都没有入口。这不是宣传语，是类型签名，`tests/test-dealer.js` 有一条测试专门钉死它（把棋盘填成快破纪录的样子、分数改到 99 万，同种子的块流必须逐块不变）。

它顺带白送了四样东西：**可验证的公平**（游戏内可查种子）、**下一手预览**（不存在"预览过的牌被偷偷换掉"）、**每日谜题**（同一天全球同一条块流，分数可比）、**撤销免疫刷块**（回滚 `streamIndex` 即可）。

⛔ **任何"让发牌看一眼棋盘/分数"的改动都会同时毁掉这四样**，并把商店页上的公平承诺变成谎言。要改先想清楚。

## 被模拟/红队/实机推翻过的判断（别再走回头路）

- **「保证至少一块可放」是假机制**：每局只触发 0.04 次。死亡几乎从不是"三块全放不下"，而是"三块里有一块放不下"（三块必须全放完才补牌）。
- **DDA（按棋盘拥挤度调权重）测不出效果**（中位分差 1.5%），却正是玩家骂 rigged 的原型（"我盘面干净时它专门发大块"）。已砍。
- **game over = 剩余的每一块都放不下**，不是"任一块放不下"——先放能放的那块，可能消掉一行腾出空间，卡住的块就又能放了。（写错过一次，玩家一眼看穿。）
- **爽点是「每一步都有回报」（streak），不是「憋大招一次多消」**。计分公式以 streak 为主引擎。
- **新手救不了于发牌**：小块权重翻倍只让新手多活 4 步，却让熟手局长爆到 10 倍。只能靠教学（前 5 关预置盘面手把手喂一次连击）。

## 关卡：三道防线堵软锁死（红队标为致命）

石块永不消 ⇒ 含石块的行/列永远清不掉。若一颗水晶的**行和列都被石块封**，它永远收集不到，而玩家又**死不了** ⇒ 无限磨、既不 win 也不 lose。

1. `levels.validate()` 构建期拦（它当场抓出过我自己写关卡时的错）
2. `core.isUnwinnable()` 运行时兜底 → 判负 + **免费重开**
3. `tools/verify-levels.js`：每关跑 200 个种子的参考 AI，**通关率 <80% 不许进包**（第一版手写的 20 关里有 7 关不达标，最差的只有 3%）

⛔ **关卡失败零广告**，只给「立刻重来」——因运气打不过的关，出口若是广告，就是我们自己在 DESIGN §9 痛骂的东西。E2E 会扫描失败界面的所有按钮，断言没有任何 AD/REVIVE。

**关卡数据别手改**：改 `tools/gen-levels.js` 的配方（水晶挂几条线 × 每条线留几个空格 × 石块数）再重新生成，然后跑 verify。
设计知识：**多条线的难度是超线性的**（4-5 线时 AI 通关率崩到 60%）；**石块是最强旋钮**（2 个石块封死 4 条线 = 通关率 -20%）。

## 本游戏踩过的坑（都真实发生过）

- **E2E 用 `dispatch()` 绕过真实点击 = 假绿**。菜单里每次点击都抛 TypeError（drag 层没查 phase），而 E2E 报"零 error"。**新加的 E2E 必须用 `page.mouse.click` 真点**。
- **顶层 `const G` 不会挂到 window 上**，而 render/drag 读的是 `window.G` → 渲染层拿到 undefined。单测永远抓不到这类跨文件全局约定问题。
- **模块顶层调用会写 `API` 的函数 = TDZ**（`Cannot access 'API' before initialization`），整个 render 模块挂掉、游戏白屏。
- **截图必须逐张肉眼验**（skill 的 HARD GATE）。它抓出了三个真实 UI bug，其中**消行预览一直是坏的**（高亮画在背景层、被实心块完全挡住）——而 DESIGN 里写着那是本作最重要的一个 UI。脚本报"生成成功"毫无意义。
- **广告红线是可测的**：E2E 断言「玩家拒绝激励视频 ⇒ 什么也不发生」（绝不强塞无奖励广告、绝不扣金币）、「局中零插屏」、「买了去广告后功能不变少」。
