## 0. 一句话

**8×8 网格，托盘给 3 块不可旋转的拼图，拖上棋盘，凑满一整行/整列即消除。** 三块全用完才补新三块；三块都放不下 = 结束。两种包装：**关卡模式**（消行时收集水晶，凑够目标数过关拿星星+金币）与**竞速模式**（无尽计分，追赶一串对手的分数线）。

## 1. 核心规则（两模式共用的引擎，先做这层）

| 项 | 规则 | 来源 |
|---|---|---|
| 棋盘 | **8×8**（截图两模式都是 8 列 × 8 行） | 截图 2/3/5-10 |
| 托盘 | 固定 **3 个槽位**，一次给 3 块 | 全部截图 |
| 补块时机 | **3 块全部放完** 才一次性补 3 块新的（不是放一块补一块）。这条是**核心张力来源**：玩家必须为「手上这三块」整体规划，而不是走一步看一步 | Block Blast 通行规则 |
| 旋转 | **不可旋转** | 截图无旋转 UI |
| 放置 | 拖拽落子（手感规格见 §4） | 截图 1、8 |
| 消除 | 落子后结算：**任意填满的整行 或 整列 全部清除**；一次同时消多条 = 高倍分（§3） | 截图 3 |
| 失败 | 托盘中**剩余的每一块都无处可放** → game over | 通行规则 |
| 计时/步数 | **无**。压力全部来自「填死自己」 | 截图无计时器/步数条 |

**唯一失败条件**：无处可放。（一条干净规则，同 mines 家风）

### 1.1 拼块池（19 种，权重分三档）

形状按格子数分档，**档位权重随棋盘拥挤度动态变化**（§2 是关键，这里只定池子）：

| 档 | 形状 | 初始权重 |
|---|---|---|
| **小**（1-3 格） | `1×1` · `1×2` · `2×1` · `1×3` · `3×1` · 小 L（3格，4 向） | 25% |
| **中**（4-5 格） | `2×2` · `1×4` · `4×1` · L/J（4格，4 向） · S/Z（4格，2 向） · T（4格，4 向） | 55% |
| **大**（5-9 格） | `1×5` · `5×1` · 十字（5格） · `2×3`/`3×2` · **`3×3`（9格，最凶）** | 20% |

- 截图里出现过的形状全在池内（3×3 实心、2×3、1×4、L、十字、S、2×2、1×2 竖条）。
- 全部走引擎 `PRNG`（可复现、可单测），**禁 `Math.random`**。
- 颜色纯装饰（消除只看行列是否填满，不看颜色）；水晶挂在具体格子上（§5）。

## 2. 发块算法（本作的命门 —— v1 只写了「反卡死救济」，这里给完整规格）

**为什么这是命门**：纯均匀随机发三块，会经常发出「刚好放不下」的组合，玩家的感受不是「我菜」而是「游戏坑我」——这是这个品类唯一的致死伤。反过来，如果永远发得下，游戏又毫无压力。**所有 Block Blast 类爆款都在偷偷做难度调节（DDA），只是不说。** 我们把它写明白、可测、可调。

### 2.1 硬约束（不变量，必须单测）

> **每次补三块后，至少有一块能放在当前棋盘上。**

实现：抽出候选三元组 → 用 `canPlaceAny(board, pieces)` 校验 → 不满足就重抽（上限 N 次，兜底强塞一个 `1×1`）。
**注意**：这条只保证「不是一发牌就死」，**不保证三块都放得下**——「三块里有一块塞不进去」正是玩家该承受的压力，别过度救济。

### 2.2 软调节（DDA：按棋盘压力调权重）

定义 `fill = 已占格数 / 64`：

| 棋盘状态 | 调节 | 意图 |
|---|---|---|
| `fill < 0.30`（太空） | 大档权重 ×1.6，小档 ×0.6 | 空盘时发小块 = 无聊白给，要让玩家早点感到"要开始规划了" |
| `0.30 ≤ fill < 0.60` | 基准权重 | 正常博弈区 |
| `fill ≥ 0.60`（拥挤） | 小档 ×2.0，大档 ×0.3，且 **3×3 直接禁发** | 拥挤时发 3×3 = 秒杀，玩家只会骂街 |
| **连续 3 回合无消除** | 直条（`1×4`/`4×1`/`1×5`/`5×1`）权重 ×2.5，持续到下次消除 | 直条是救场件；给玩家一个"翻盘的机会"而不是慢性死亡 |

**期望可放数**（软目标，不做硬约束、只做测试断言的统计口径）：跑 1000 局随机 AI，**「三块全放得下」的回合占比应落在 60%~80%**。低于 60% = 玩家会觉得被坑；高于 80% = 没压力。这是一个**可测量的手感指标**，调 DDA 就调它。

### 2.3 反作弊式的"公平"

⚠ **绝不做「玩家快赢了就发死块」的恶意调节**（部分同类产品这么干，是差评来源）。DDA 只看**棋盘几何**，**不看**玩家分数/是否接近过关/是否付过费。这条写进代码注释和测试。

## 3. 计分与 Combo（v1 全是 TBD，这里给公式）

```
落子分   = 拼块格子数                          // 1×1 得 1，3×3 得 9
消除分   = 10 × L × (L+1) / 2 × comboMult      // L = 本次同时消除的行列总条数
           L=1 → 10   L=2 → 30   L=3 → 60   L=4 → 100   L=5 → 150
comboMult = min(1 + 0.5 × (streak - 1), 4.0)   // streak = 连续「有消除」的回合数
           连消 1/2/3/4/…回合 → ×1 / ×1.5 / ×2 / ×2.5 …  上限 ×4
```

- **一回合没消除 → streak 归零**。这制造了「宁可绕路也要保住连击」的决策张力，是分数上限的主要来源。
- 数量级自检：截图里一局打到 1703 分，用本公式 ≈ 40-60 回合、平均每回合一条多、偶有 2-3 连消 —— 量级吻合。
- **L 的非线性是爽点的数学来源**：一次消 3 条（60）＞ 分三次各消 1 条（30）。所有高手行为（憋大招、凑十字消）都由这条公式长出来，不需要额外设计。

## 4. 拖拽手感（⚠ 引擎缺口 + 具体参数）

`engine/input.js` 只有 **tap → hitTest → onAction** 和 **swipe**，**没有 drag & drop**。本仓 4 个已上线游戏全是点击/滑动类，本作是第一个真拖拽游戏。

- **做法**：drag 层写在 `games/blockblast/js/drag.js`（游戏自己监听 pointerdown/move/up），**不动 `engine/input.js`** —— 改 engine 会牵动 4 个线上游戏且必须跑全量测试（root CLAUDE.md 明写的协作坑）。等第二个拖拽游戏出现再抽取到 engine。
- **参数（移动端必须这样，否则手指盖住拼块）**：

| 参数 | 初值 | 说明 |
|---|---|---|
| 抬起偏移 | 拼块中心上移 **1.2 格** | 手指按住时拼块浮在指尖上方，能看见自己在放什么。**不做这个 = 移动端不可玩** |
| 拾起放大 | 1.0 → **1.1×**，80ms | 手感反馈：块"活了" |
| 吸附 | 拼块**锚点格**（左上角格）中心落在哪格就吸哪格；无阈值容差，全靠抬起偏移保证可见 | 简单可预测 > 聪明 |
| 幽灵预览 | 合法：目标格半透明白 + **将被消掉的整行/整列整条发光高亮**（截图 8 那条黄色列）；非法：拼块变暗红、不显示幽灵 | **消行预览是本作最重要的一个 UI**：它把"我这一步能消行"从心算变成看见，直接决定新手能否上手 |
| 回弹 | 非法松手 → 180ms ease-out 弹回托盘原位 | 别瞬移，玩家要看清"没放上" |

## 5. 模式 A：关卡收集（P2，先做）

对应截图 1–4。

- **目标条**（棋盘上方）：1–3 种收集物 + 需求数量（截图 2：`💎7 / 🔷3 / 🔺3`）。达成的项打绿勾 ✔（截图 3）。全达成 → 立即胜利。
- **水晶两个来源**：① 关卡预置（开局棋盘上就有带水晶的方块）；② 托盘拼块自带（截图 2 的蓝色拼块上有橙三角），放上棋盘后随格子留在盘上。
- **收集时机**：**只有该格被消行/消列清除时**才计入（教程原话「Burn lines and collect crystals」）。水晶格 = 「必须打通那一行/列」的定点目标 —— **这是关卡模式的全部乐趣来源**，也是它与竞速模式的唯一玩法差异：竞速是「随便消哪条都行」，关卡是「必须消那一条」。
- **关卡难度曲线**（30 关首批，数据表驱动 `levels.js`）：

| 关段 | 引入 | 意图 |
|---|---|---|
| 1-3 | 空盘 + 1 种水晶 ×3，水晶都在同一行 | 教「消行才收集」这一条 |
| 4-8 | 2 种水晶，分散到不同行列 | 教「要规划先打哪条」 |
| 9-15 | 预置阻塞块（无水晶的死格） | 教「盘面不是空的，空间要省」 |
| 16-24 | 3 种水晶 + 水晶藏在角落/边列 | 边列最难填满 = 真正的难点 |
| 25-30 | 石块障碍（§5.1）+ 大量预置 | 收尾难度 |

- **胜利结算**（截图 4）：`Excellent Game!` + 金币 + **星星**，`Tap to continue`。
- **失败**：无处可放 → 「重来 / 看广告换一批托盘继续」（§8）。

### 5.1 灰色石块（截图 9，v1 标了「待确认」—— v2 给它一个设计）

无论原作是什么，**我们定义为**：**石块 = 不可消除的惰性格**，占位、不参与消行判定的"填满"（即一行里有石块 → 那行永远消不掉）。它作为**关卡障碍**从棋盘预置出现（不进托盘），玩家必须绕开它规划。
理由：这给了关卡设计师一个强力的空间约束工具，且规则依然只有一条（"石块不消失"），不增加认知负担。
（截图 9 里它出现在**托盘**，若实机确认确实是"可放置的石块拼块"，那它就是一个**惩罚块**——必须放下、永久占格。这更狠也更有趣，作为 §12 的实机确认项。）

### 5.2 三星评级（v1「待定」→ v2 定下来：做）

**每关按剩余空格数评星**：过关瞬间 `empty = 空格数`。`empty ≥ 24` → ⭐⭐⭐；`≥ 12` → ⭐⭐；否则 ⭐。
理由：① 给了**重玩理由**（纯克隆最缺的就是重玩动机）；② 评价维度是「打得干净利落」而不是「运气好」，与技巧正相关；③ 星星喂给章节宝箱（§9），形成进度感闭环。截图只见「星星 ×1」，但那很可能就是一星结算。

## 6. 模式 B：对手竞速（P3）

对应截图 5–10。无尽计分 + 一串对手追赶；棋盘/托盘/消除规则与模式 A 完全相同，只换外壳。

- 顶部**大黄数字** = 待超越的分数线，**旗帜 🏁 数字** = 当前局得分（0 → 359 → 853 → 1027 → 1703）。超过当前对手 → 右侧头像换下一个（熊 → 草莓 → 金发女 → …），进度条绿色段前进一截。
- **判定为本地 bot 队列，不需要服务器**：一条递增的分数线表（如 `[300, 600, 900, 1400, 2000, 2800, …]`，末尾按公式外推到无限），每个分数线配一个头像+名字。玩家超过就前进。
- 设计意图：把「无尽模式的空虚计分」切成**一连串 30 秒兑现的小胜利** —— 这是无尽模式留存的关键，比单纯的最高分强得多。
- 无失败罚：卡死即结束，进度条到哪算哪，结算给金币。

## 7. 消行反馈 / Juice（v1 完全没写 —— 但它就是这个品类的产品本体）

**残酷的事实**：block puzzle 的规则谁都会写，玩家留下来是因为**消行的那 0.4 秒手感**。这一节的优先级 **等同于核心逻辑**，不是"美术二期再说"。

| 时刻 | 反馈 |
|---|---|
| 落子 | 方块"砸"下去：轻微下压 + 落地弹一下（scale 1.05 → 1.0, 100ms）+ 短促"啪"音 + 轻震动 |
| 消行判定 | **逐格延迟消失**（从消除中心向两端扩散，每格延迟 20ms）—— 而不是整行同时消失。这一条差别巨大：扩散感 = "我引爆了它" |
| 粒子 | 每个消失的格子迸出 3-5 个同色碎片，重力下落淡出 |
| 多条同消 | 屏幕轻震（幅度随 L 递增）+ 屏幕中央弹出 `DOUBLE! / TRIPLE!` 大字 |
| Combo | 音效**音高随 streak 上行**（C-D-E-F-G… 音阶），这是最便宜也最上瘾的正反馈；`COMBO ×2.5` 字样 |
| 收集水晶 | 水晶从消失的格子**飞向顶部目标条**（贝塞尔曲线，300ms），到达时目标条数字跳动 + "叮" |
| 目标达成 | 该项打绿勾 + 弹一下 |
| 卡死前 | 当**只剩一块可放**时，可考虑不提示（保持紧张）；game over 时把"放不下"的三块红色抖动一下，让玩家看见"确实放不下" |

所有动画走 `gameMs`（引擎时钟），**禁 `Date.now()`**；切后台冻结。

## 8. 道具 / 经济 / 变现

底部两个道具（截图 2/3/5-10）：

| 道具 | 作用 | 获取阶梯（截图证据 + 我们的定值） |
|---|---|---|
| **刷新** Refresh | 弃掉托盘 3 块，换一组新的 | **每 8 次落子充能 1 格**（截图 2 灰=没电、截图 3 绿=有电）；充能满时可 `📺 FREE` 看广告立即用 |
| **撤销** Undo | 撤回上一步落子（含被消行的复原、水晶收集数回退、streak 回退） | `×1` 本局免费（截图 5）→ `📺 FREE` 看视频（截图 6-9）→ **金币**（截图 10 是 1300；我们定 **300**，1300 是明显的"逼氪价"，我们不装这个坑） |

- **金币**：左上角显示，带 `+` → 商店（看视频领币；IAP 二期）。来源：关卡结算（20-40，按星数）、竞速结算、广告。用途：买撤销、买刷新。
- **激励视频**（`Ads.rewarded`）：刷新 / 撤销 / **失败后续命**（换一批托盘继续，每局限 2 次）/ 领金币。
- **插屏**（`Ads.interstitial`）：关卡结算「Tap to continue」之后、**每 3 关一次**；**绝不打断局内**。
- 每游戏自备 adUnits，**绝不复用他游戏的**（引擎契约）。

## 9. 进度与留存（v1 缺失 —— 纯克隆最容易漏的一层）

- **章节地图**：30 关分 3 章，每章末一个**星星宝箱**（凑够 X 星开启）→ 金币 + **皮肤**。这是三星评级的兑现出口。
- **皮肤**（薄差异化，本仓家风）：方块主题 4 套（糖果 / 宝石 / 像素 / 深海），`PAL` 调色板切换引用，**渲染函数必须确定性、禁 `Math.random`**（snake 实踩）。
- **竞速最高分** + 已超越的对手数（"你已击败 17 位玩家"）。
- **成就**（薄，~30 条，数据表驱动）：单局最高 combo、一次四消、零撤销通关、连续 7 天…
- **只存**：关卡进度/星星/金币/皮肤/成就/最高分/当局快照。**不存永久数值加成**（不给"永久 +10% 分数"这种，会毁掉分数的可比性）。

- 顶栏：金币 + `+`（左）；**退出**（红）、**设置**（蓝，音效/音乐/震动/语言）（右）。
- 主菜单（截图未覆盖，自设计）：章节地图（主）+ 竞速模式入口 + 商店 + 设置。

## 11. 工程（严守引擎契约，见 `engine/README.md`）

### 11.1 模块分层（index.html 加载顺 = 依赖顺，load-bearing）

引擎脚本 → `prng` → `pieces`（形状表+权重）→ `dealer`（**§2 发块/DDA，单独一层，因为它是命门且要重度单测**）→ `levels`（关卡表）→ `bots`（竞速分数线）→ `core`（棋盘/放置判定/消除/计分/combo/game-over/收集/撤销栈）→ `storage` → `drag` → `fx`（§7 粒子/动画队列）→ `render` → `main`。

- **纯逻辑层**（双导出 `if(module.exports)…else 全局`，node 可单测）：`prng / pieces / dealer / levels / bots / core`。改这些**先写测试**。
- **DOM/渲染层**：`render`（`renderAll()` 契约：每帧 `clearHits()` 全屏重画 + `addHit()`）、`drag`、`fx`、`main`。
- **状态**：单一可变 `G`（`G.phase`：menu / playing / dragging / win / lose）；所有交互进 `dispatch(action, data)`。
- **撤销栈**：`core` 维护 `history[]`（每步存：落子前棋盘快照 + 分数 + streak + 收集进度 + 托盘）。**只需存 1 步**（撤销只回退一步），避免存档膨胀。
- **零硬编码文案**：`T('blockblast.xxx')` + `locales/<lang>.json`（**必须嵌套结构** —— 扁平写法查不到且零报错，snake 实踩；`check-locales` 查不出这个）。en 基准 + zh-CN 起步。canvas 非定长文案过 `wrapLines`。
- **存档**（`CFG.key('run')`，带 `v: N`）：进度/星星/金币/道具/皮肤 + 当局快照（棋盘/托盘/分数/streak/目标进度/DDA 状态/PRNG 种子）支持中途关页续玩。**改 `G` 形状必 bump `SAVE_VERSION`（旧档丢弃不迁移）；改任何 js/css 必 bump index.html 所有 `?v=N`。**

### 11.2 测试

- **node 单测**（无 DOM，核心）：
  - 放置合法性、消行/消列/同时多条、**计分公式与 combo 倍率的边界**（streak 归零、×4 封顶）；
  - **game-over 判定 = 且仅当三块都放不下**；
  - **`dealer` 硬约束：补块后至少一块可放**（跑 10k 次随机棋盘）；
  - **`dealer` 不看玩家分数**（把分数/关卡进度传进去，断言输出与不传时完全一致 —— §2.3 的公平性可测化）；
  - 水晶**只在消除时**计入；撤销能**精确复原**（棋盘/分数/streak/收集/托盘全一致）；
  - PRNG 可复现（同种子同序列）。
- **auto-player 冒烟**：贪心 AI 跑 1000 局 → 断言不崩、无 NaN、棋盘形状不变、分数单调不减，并**统计 §2.2 的「三块全可放回合占比」落在 60-80%**（手感回归测试）。
- **E2E（playwright 无头）**：拖拽落子（pointer 事件序列）、消行、过关结算、三星、道具三态、存档续玩、语言切换。
