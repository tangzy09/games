# blockblast —— 设计/需求文档

> **内部代号 `blockblast`（目录名），⛔ 绝不能当商店名** —— *Block Blast!* 是 Hungry Studio 的注册商标+已上架爆款，用它上架 = App Store 4.3/商标下架，同 abyssshoot 的命名红线。商店名二期另取原创名（候选：Cube Burst / Gem Drop Puzzle …）。
>
> **玩法定位（用户已定）**：**纯克隆 Block Blast**，忠实还原参考截图的核心循环与两套模式；不做肉鸽抽卡。差异化只留一层薄的收集/皮肤（本仓家风），肉鸽若做也是二期的第三个模式，不动主玩法。
>
> **两套模式，关卡优先**：先做核心消除 + 关卡收集模式，再做对手竞速模式。
>
> root `CLAUDE.md`（引擎契约、部署铁律、git 纪律、iOS 流水线）与 `engine/README.md` 先读，本文件只讲本游戏专属设计。

## 0. 一句话

**8×8 网格，托盘给 3 块不可旋转的拼图，拖上棋盘，凑满一整行/整列即消除。** 三块全用完才补新三块；三块都放不下 = 结束。两种包装：**关卡模式**（消行时收集水晶，凑够目标数过关拿星星+金币）与**竞速模式**（无尽计分，追赶一串对手的分数线）。

## 1. 核心玩法（两模式共用的引擎，先做这层）

| 项 | 规则 | 来源 |
|---|---|---|
| 棋盘 | **8×8**（截图两模式都是 8 列 × 8 行） | 截图 2/3/5-10 |
| 托盘 | 固定 **3 个槽位**，一次给 3 块 | 全部截图 |
| 补块时机 | **3 块全部放完** 才一次性补 3 块新的（不是放一块补一块） | Block Blast 通行规则 |
| 旋转 | **不可旋转**（无旋转按钮，拼块形状即最终形状） | 截图无旋转 UI |
| 放置 | 拖拽：按住拼块 → 跟随手指 → 网格吸附 + **幽灵预览**（合法位置高亮；将被消掉的整行/整列**整条高亮**，见截图 8 那条发光的黄色列）→ 松手落子；非法位置松手 = 弹回托盘 | 截图 1（手指引导）、截图 8（高亮列） |
| 消除 | 落子后结算：**任意填满的整行 或 整列 全部清除**；一次同时消多条 = combo 加分 | 截图 3 |
| 失败 | 托盘中**剩余的每一块都无处可放** → game over（关卡=失败，竞速=结束本局） | 通行规则 |
| 计时/步数 | **无**。既不限时也不限步，压力全部来自「填死自己」 | 截图无计时器/步数条 |

**唯一失败条件**：无处可放。（一条干净规则，同 mines 家风）

### 1.1 拼块池

- 形状族：1×1 … 1×5 / 2×5 直条，2×2、3×3 方块，L/J/T/S/Z/十字 等多联骨牌变体（截图见：3×3 实心、2×3、1×4、L 型、十字、S 型、2×2、单格竖条）。
- **加权随机**抽取，全部走引擎 `PRNG`（可复现、可单测），**禁 `Math.random`**。
- ⚠ **反卡死救济（重要设计旋钮，需实机调）**：纯随机三块会频繁「刚好放不下」，体感极差。策略：抽取时对**当前棋盘做可放性检查**，保证新的一组 3 块中**至少 1 块当前可放**（更宽松/更严格由难度曲线定）。这是本作手感的命门，单测必须覆盖「补块后至少一块可放」这一不变量。
- 颜色：纯装饰（消除不看颜色，只看行列填满），但关卡的水晶挂在具体格子上。

### 1.2 计分（竞速模式主用，关卡模式也累计）

- 落子分 = 拼块格子数。
- 消除分 = 每消一条基础分 × **同回合消除条数的连乘/递增系数**（消 1 条 vs 一次消 3 条差距要大，这是爽点）。
- **Combo/连击**：连续多个回合都有消除 → 倍率递增；有一回合没消除则清零（Block Blast 招牌，需实机验证倍率曲线）。

## 2. 模式 A：关卡收集（P2，先做）

对应截图 1–4。

- **目标条**（棋盘上方）：1–3 种收集物 + 需求数量，如 `💎7 / 🔷3 / 🔺3`（截图 2）。达成的那项打绿勾 ✔（截图 3）。全部达成 → 立即胜利。
- **收集物 = 嵌在方块里的水晶**，两个来源：
  1. **关卡预置**：开局棋盘上就摆着若干带水晶的方块（截图 2 的红块带蓝钻、黄块带粉宝石）；
  2. **托盘拼块自带**：抽到的拼块某些格子上带水晶（截图 2 托盘的蓝色拼块上有橙三角），放上棋盘后随格子留在盘上。
- **收集时机**：**只有该格子被消行/消列清除时**才计入收集（「Burn lines and collect crystals」——截图 1 教程原话）。所以水晶格是「必须打通那一行/列」的定点目标，这是关卡模式的全部乐趣来源。
- **障碍块（待确认）**：截图 9 托盘里出现**灰色 3×3 石块**。推测为「不带颜色的惰性块」或某种障碍/惩罚块（也可能只是别的模式的皮肤）。**先按普通拼块实现，机制留待实机确认**（见 §11 不确定清单）。
- **胜利结算**（截图 4）：横幅 `Excellent Game!` + **金币 ×N**（截图 32）+ **星星 ×1**，`Tap to continue`。
  - 星星用途：暂定「每关 1 星，星星累积解锁章节宝箱」；**是否做三星评级待定**（截图只见 ×1）。
- **失败**：无处可放 → 弹「再来一次 / 看广告加一次刷新续命」（见 §4）。
- **关卡数据**：`levels.js` 数据表驱动（棋盘预置格 + 水晶分布 + 目标 + 拼块池权重 + 奖励），**不写死判定逻辑**。首批 ~30 关手工设计，后续可生成。

## 3. 模式 B：对手竞速（P3）

对应截图 5–10。**无尽计分 + 一串对手追赶**，棋盘/托盘/消除规则与模式 A 完全相同，只换外壳。

- 顶部：**大黄数字** = 本局的目标线/待超越分（截图 5 显示 713，此时本局才 0 分），下方**旗帜 🏁 数字** = **当前局得分**（0 → 359 → 853 → 884 → 1027 → 1703）。
- 一超过当前对手，大黄数字随之跟到当前分（截图 7 起两者相等），**右侧对手头像换成下一个**（熊 → 草莓 → 金发女 → …，截图 5/6/7/10 各不同），进度条的绿色段前进一截。
- 解读：**一串排队的 bot「对手」，每超越一个就换下一个、进度条推进**，本质是节奏化的成就/追赶反馈，不是真 PvP（**不需要服务器**，bot 分数线本地表驱动即可）。
- ⚠ 这段是从静态截图**推断**的，实机确认后修正（见 §11）。
- 无失败罚：卡死即结束，进度条到哪算哪，结算给金币。

## 4. 道具 / 经济 / 变现（贯穿两模式）

底部两个道具按钮（截图 2/3/5-10）：

| 道具 | 作用 | 定价阶梯（截图证据） |
|---|---|---|
| **刷新 Refresh**（白色旋转箭头） | 弃掉当前托盘 3 块，换一组新的 | 有**冷却/充能**：截图 2 灰（不可用）→ 截图 3 绿（可用）。可用时标 `📺 FREE` = **看激励视频免费用** |

## 6. 工程（严守引擎契约，见 `engine/README.md`）

### 6.1 ⚠ 引擎缺口：拖拽（本作最大的技术新增）

`engine/input.js` 只有 **tap → hitTest → onAction** 与 **swipe**，**没有 drag & drop**（按下-跟随-放开三段式 + 移动中实时预览）。本仓现有 4 个游戏都是点击/滑动，本作是第一个真拖拽游戏。

- **做法**：drag 层**先写在 `games/blockblast/js/drag.js` 内**（游戏自己监听 pointerdown/move/up 或复用 `Input` 的 liveSwipe 通道），**不动 `engine/input.js`**——改 engine 会牵动 4 个已上线游戏且必须跑全量测试（root CLAUDE.md 明写的协作坑）。第二个拖拽游戏出现时再回头抽取到 engine。
- **手感要点**：手指按住时拼块要**上移一段偏移**（否则被手指盖住，移动端必踩）；吸附到最近合法格；非法时预览用红/暗色区分；松手非法 → 回弹动画。

### 6.2 模块分层（index.html 加载顺 = 依赖顺，load-bearing）

引擎脚本 → `prng` → `pieces`（拼块形状表/权重）→ `levels`（关卡数据表）→ `bots`（竞速对手分数线）→ `core`（棋盘/放置判定/消除/计分/combo/game-over/收集结算）→ `storage` → `drag`（拖拽层）→ `render`（renderAll + 幽灵预览 + 高亮）→ `main`（boot/事件/浮层/存档）。

- **纯逻辑层**（双导出 `if(module.exports)…else 全局`，node 可单测）：`prng / pieces / levels / bots / core`。改这些**先写测试**。
- **DOM/渲染层**：`render`（`renderAll()` 契约：每帧 `clearHits()` 全屏重画 + `addHit()`）、`drag`、`main`。
- **状态**：单一可变 `G`（含 `G.phase`：menu / playing / dragging / win / lose）。所有交互进 `dispatch(action, data)`。
- **零硬编码文案**：全走 `T('blockblast.xxx')` + `locales/<lang>.json`（**必须嵌套结构**——扁平写法查不到且零报错，snake 实踩；`check-locales` 查不出这个）。en 基准 + zh-CN 起步。canvas 非定长文案过 `wrapLines`。
- **存档**（`CFG.key('run')`，带 `v: N`）：关卡进度/星星/金币/道具次数 + 当局快照（棋盘/托盘/分数/combo/目标进度）支持中途关页续玩。**改 `G` 形状必 bump `SAVE_VERSION`，旧档丢弃不迁移；改任何 js/css 必 bump index.html 所有 `?v=N`。**

### 6.3 测试

- **node 单测**（核心，无 DOM）：放置合法性、消行/消列/同时多条、combo 与计分、**game-over 判定**（三块全放不下 = 且仅当）、**补块「至少一块可放」不变量**、水晶收集只在消除时计入、撤销能精确复原（含被消掉的行）、PRNG 复现。
- **auto-player 冒烟**：随机/贪心 AI 跑 N 局，断言不崩、无 NaN、棋盘形状不变、分数单调不减。
- **E2E（playwright 无头）**：拖拽落子（pointer 事件序列）、消行、过关结算、道具三态（免费/视频/金币）、存档续玩、语言切换。

## 8. 分阶段实施

1. **P1 核爽**（先只有竞速的无尽计分壳，验手感）：8×8 棋盘 + 拼块池 + **拖拽层** + 放置/消除/计分/combo/game-over + 反卡死抽块 + node 单测 + 无头冒烟。**先确认「拖拽吸附 + 消行爽感 + 卡死压力」成立**再往下。
2. **P2 关卡模式**：水晶收集（预置 + 拼块自带）+ 目标条 + 30 关数据表 + 星星/金币结算 + 新手引导。
3. **P3 竞速模式**：bot 对手队列 + 进度条 + 超越换头像 + 最高分。
4. **P4 元层 + 变现 + 上架**：道具三态（免费/视频/金币）+ 商店 + Ads 钩子 + i18n 十语 + 皮肤/收集（薄）+ iOS 壳（`package.json`/`capacitor.config.json`/`resources/` + codemagic workflow 4 vars）+ 商店素材 + **原创商店名**。
