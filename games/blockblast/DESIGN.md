## 0. 一句话

**8×8 网格，托盘给 3 块不可旋转的拼图，拖上棋盘，凑满一整行/整列即消除。** 三块全用完才补新三块；三块都放不下 = 结束。两种包装：**关卡模式**（消行时收集水晶，凑够目标过关）与**竞速模式**（无尽计分，追赶一串对手）。

## 0.1 调研的四个结论（v3 的全部依据）

| # | 结论 | 对设计的强制要求 |
|---|---|---|
| **1** | **爽点是「每一步都有回报」，不是「憋大招」。** 玩家原话：「无脑，但**每一步都有回报**」。攻略圈共识：**连续 5 步各消 1 条 ≫ 一步消 5 条再空放 4 步**。真正的引擎是 **streak（连续回合都有消除）**，多条同消只是锦上添花。 | **§3 计分公式以 streak 为主引擎**（v2 搞反了，已重写） |
| **2** | **这个品类的第一杀手是广告，不是发块。** Block Blast 的 Metacritic 用户评 **5.2/10、45% 负评**，第一抱怨是广告；最招恨的是「**拒绝复活广告后仍强塞一个无奖励广告**」；Woodoku 被骂多年的是「**没有一次性去广告 IAP，我想付钱都不行**」。 | **§9 变现三条红线 + 必做去广告 IAP** |
| **3** | **玩家的公平感来自「可解释的失败」，不是真随机。** 「rigged」是全品类行话（无一份可信实测证实，但人人相信）。转投 Woodoku 的玩家说它更公平，理由是 3×3 宫逼你算两步，**输了觉得"是我菜"**。最扎心的抱怨：「**下一块永远是我上一步需要的那块**」——晚一步给比不给更抓狂。 | **§2 把公平做成可感知、可验证的机制**（不是靠公关辩解） |
| **4** | **perfect clear（清空整盘）是无人占领的空位。** Block Blast / Blockudoku / Woodoku 的计分里**都没有**清盘奖励；而 Tetris 的 perfect clear 玩家极度追捧（10× 乘数、专门的 PCO 开局套路）。 | **§4 把「清空盘面」做成本作招牌大招** ——这是最便宜的一刀差异化 |

**另外两条基调**（调研共识，写死不动）：

- ⛔ **永不加倒计时/下落压力**。玩家把它当**睡前/摸鱼的放松游戏**（"每晚睡前在床上玩"、"边听音乐边玩"）。相对 Tetris 的核心优势就是**块在托盘里等你**。烧脑感必须来自玩家自己追高分，**不能由系统施压**。
- 📉 **真实数据口径**（第三方，非厂商 PR）：D1 ≈ 26%、**单次会话 ≈ 12 分钟、每天 2.7 次**。→ **单局设计目标 3-5 分钟，一次会话连打好几局**。这是靠体量而非深度的品类，别做重系统。

## 1. 核心规则

| 项 | 规则 | 依据 |
|---|---|---|
| 棋盘 | **8×8** | Block Blast 官方描述 + 所有克隆一致（9×9+3×3 宫是 Blockudoku 路线，会多一条消除轴、深度更高但变成另一个品类——**不走**） |
| 托盘 | 3 个槽位，一次给 3 块 | 全部截图 |
| 补块 | **3 块全放完**才补新的 3 块 | 通行规则；这是核心张力：必须为「手上这三块」整体规划 |
| 旋转 | **不可旋转** | 截图无旋转 UI |
| 放置 | 拖拽（手感规格 §5） | 截图 1、8 |
| 消除 | 填满的整行/整列全清 | 截图 3 |
| 失败 | 剩余的每一块都无处可放 | 通行规则 |
| 计时 | **无**（见 §0.1 基调） | 品类共识 |

### 1.1 拼块池（19 种）

| 档 | 形状 | 基准权重 |
|---|---|---|
| **小**（1-3 格） | `1×1` · `1×2`/`2×1` · `1×3`/`3×1` · 小 L（3 格 ×4 向） | 25% |
| **中**（4-5 格） | `2×2` · `1×4`/`4×1` · L/J（4 格 ×4 向） · S/Z · T | 55% |
| **大**（5-9 格） | `1×5`/`5×1` · 十字 · `2×3`/`3×2` · **`3×3`（最凶）** | 20% |

截图出现过的形状全在池内。全部走引擎 `PRNG`（可复现、可单测），**禁 `Math.random`**。颜色纯装饰（消除只看行列填满）。

## 2. 发块算法与「可感知的公平」（§0.1 结论 3）

**问题的本质**：原版 Block Blast **不保证**至少一块可放（死亡是它的广告变现钩子），于是「rigged」成了全品类行话——玩家坚信游戏在接近纪录时发死块。没有任何可信实测证实这一点（那些声称"逆向了 50 万次发块"的站点是 SEO 垃圾站，数据系编造），但**玩家的感受是真实的产品问题：你无法用"其实是随机的"说服他们**。

**我们的路线：不辩解，把公平做成机制，并且让玩家能验证。**

### 2.1 三条硬保证（每条都可单测、且写进游戏内的说明页）

1. **每次补三块，保证至少一块能放在当前棋盘上。**（抽候选组 → `canPlaceAny()` 校验 → 不满足重抽，兜底强塞 `1×1`）
   *但不保证三块都放得下* —— 「三块里有一块塞不进去」正是玩家该承受的压力。
2. **发块只看棋盘几何，绝不看玩家的分数 / 是否接近纪录 / 是否接近过关 / 是否付过费。**
   → **可测化**：把分数、关卡进度、付费状态传进 `dealer`，断言输出与不传时**逐字节一致**。这条测试是本作的良心。
3. **绝不做「记忆式惩罚」**——即绝不实现"记住玩家上一步需要什么、这一步才给"（玩家最扎心的那条抱怨：「下一块永远是我上一步需要的」。**晚一步给比不给更抓狂**）。`dealer` 无历史需求记忆，只有棋盘几何 + PRNG。

### 2.2 软调节（DDA，只看棋盘几何）

定义 `fill = 已占格数 / 64`：

| 状态 | 调节 | 意图 |
|---|---|---|
| `fill < 0.30` | 大档 ×1.6，小档 ×0.6 | 空盘发小块 = 无聊白给 |
| `0.30 ≤ fill < 0.60` | 基准 | 正常博弈区 |
| `fill ≥ 0.60` | 小档 ×2.0，大档 ×0.3，**3×3 禁发** | 拥挤时发 3×3 = 秒杀，玩家只会骂街 |
| 连续 3 回合无消除 | 直条（`1×4`/`4×1`/`1×5`/`5×1`）×2.5，至下次消除 | 救场件，给翻盘机会而非慢性死亡 |

**可测量的手感指标**：跑 1000 局随机 AI，**「三块全放得下」的回合占比应落在 60%~80%**。低于 60% = 玩家觉得被坑；高于 80% = 没压力。调手感就调这个数。

### 2.3 让公平「可被看见」（差异化，成本极低）

- **下一组预览**：显示下一组 3 块（至少显示第 1 块）。调研里玩家把「**只给看 3 块、无法为将来留位**」明确列为不公平来源。给了预览，玩家就能规划两步，**失败随之变得可归因**——这正是 Woodoku 被认为更公平的原因（它靠 3×3 宫达到同样效果）。可在设置里关掉（硬核模式）。
- **游戏内「公平宣言」页**（设置里一条）：白纸黑字写明上面三条硬保证。头部产品没人敢写，我们敢——**这是能直接进商店描述和 ASO 的卖点**（"No rigged blocks. Ever."）。
- **可归因的失败**：game over 时，把三块**逐一在棋盘上红色扫一遍**（演示"确实每格都放不下"），而不是一个干巴巴的 "No moves"。玩家要看见自己是**死于自己的填法**，不是死于发牌。

## 3. 计分：以 streak 为主引擎（⚠ v2 搞反了，这里是重写）

**v2 的错误**：把「一次消多条的非线性」当成爽点的数学来源。**调研结论 1 说明恰恰相反**——真正让人上瘾的是「每一步都有回报」，即 **streak：连续每个回合都有消除**。系统应该逼玩家**每一步都消掉点什么**，而不是憋大招。

```
落子分   = 拼块格子数                                   // 1×1 得 1，3×3 得 9

消除分   = 20 × L × comboTier(L) × streakMult
  L = 本次落子同时消掉的行+列总条数
  comboTier: L=1 → 1.0 | L=2 → 1.5 | L=3 → 2.2 | L≥4 → 3.0    // 多消有加成，但不是主菜
  streakMult = 1 + 0.25 × min(streak - 1, 12)                  // 上限 4.0，慢热
  streak = 连续「有消除」的回合数

streak 断裂 = 连续 2 个回合零消除   // ⚠ 不是 1 个回合就断
```

**三个刻意的设计决定**：

1. **streakMult 慢热（每回合只 +0.25，要 13 连才封顶 ×4）**：让「一直保持消除」成为长期目标，而不是几步就到顶。这条曲线是本作的分数上限所在。
2. **断裂需要连续 2 回合零消除**（原作克隆 blockerino 是「一整轮托盘 3 手无消除才断」，Blockudoku 是 1 回合就断）。取中间值：**给玩家一个"垫一步"的容错**，这直接对冲了"被发块坑"的挫败感——手气差的一回合不会立刻毁掉苦心经营的连击。
3. **对照真实逆向公式做过量级校验**：Blockudoku 引擎（gary-z/blokie 逆向）是「1 分/格 + 每线 18 分 × combo 分档（18/36/54/72）+ streak 固定 +9」；1010! 是 `5r(r+1)`。我们的结构与之同构，但**把 streak 从"固定加分"升级成"乘数"**——因为调研说 streak 才是爽点，就该让它主导分数。用本公式回算截图那局 1703 分 ≈ 40-60 回合、平均每回合一条多、偶有 2-3 连消，量级吻合。

## 4. Perfect Clear：本作的招牌大招（§0.1 结论 4 —— 无人占领的空位）

> Block Blast / Blockudoku / Woodoku 的计分里**全都没有清盘奖励**。而 Tetris 的 perfect clear 被玩家极度追捧（有版本给 10× 乘数，甚至发展出专门的开局套路）。**这是一个免费的、被验证过的爽点，却没人在这个品类里做。**

**规则**：一次落子后，若**棋盘变为完全空**（`fill == 0`）→ **PERFECT CLEAR**。

| 奖励 | 值 |
|---|---|
| 分数 | **本次消除分 × 5**（外加固定 500） |
| streak | **不断，且直接 +3**（奖励而非惩罚清盘后的空窗） |
| 特效 | 全屏白闪 → 棋盘边框金光爆开 → 大字 **PERFECT!** + 专属音效（比任何 combo 音效都高一个八度）+ 长震动 |
| 金币 | +10 |
| 成就/统计 | 「本局 perfect clear 次数」独立计数，主菜单显示历史最高 |

**为什么它能成为招牌**：
- 它给了空盘一个**目标**（原本清空盘面只是"状态好"，现在是"值得追求的事件"）；
- 它创造了一整套**高手玩法**（为了清盘而故意留少量块、算清盘路径）——纯克隆最缺的就是技巧天花板；
- 它是**可以直接放进商店截图和买量素材的画面**；
- 实现成本：约 30 行代码 + 一个特效。

**关卡模式里**：perfect clear 可作为**支线目标**（"清空盘面 1 次"），也是三星条件之一（§6.2）。

## 5. 拖拽手感（⚠ 引擎缺口）

`engine/input.js` 只有 tap 和 swipe，**没有 drag & drop**；本仓 4 个已上线游戏全是点击/滑动类，本作是第一个真拖拽游戏。

- **做法**：drag 层写在 `games/blockblast/js/drag.js`（游戏自己监听 pointerdown/move/up），**不动 `engine/input.js`** —— 改 engine 会牵动 4 个线上游戏且必须跑全量测试（root CLAUDE.md 明写的协作坑）。第二个拖拽游戏出现时再抽取。

| 参数 | 初值 | 说明 |
|---|---|---|
| 抬起偏移 | 拼块中心**上移 1.2 格** | 手指按住时拼块浮在指尖上方。**不做这个 = 移动端不可玩** |
| 拾起放大 | 1.0 → **1.1×**，80ms | 块"活了" |
| 吸附 | 锚点格中心落在哪格就吸哪格，无容差 | 可预测 > 聪明 |
| 幽灵预览 | 合法：目标格半透明白 + **将被消掉的整行/整列整条发光**（截图 8）；非法：拼块暗红、不显示幽灵 | **消行预览是本作最重要的一个 UI**——它把"这步能消行"从心算变成看见 |
| 回弹 | 非法松手 → 180ms ease-out 弹回 | 让玩家看清"没放上" |

## 6. 模式 A：关卡收集（P2，先做）

对应截图 1–4。（原版 Block Blast 的 Adventure 模式确为"收集宝石/限定目标"，与截图一致。）

- **目标条**：1–3 种收集物 + 数量（截图 2：`💎7 / 🔷3 / 🔺3`），达成打绿勾 ✔。全达成 → 胜利。
- **水晶来源**：① 关卡预置（棋盘上带水晶的方块）；② 托盘拼块自带（截图 2）。
- **收集时机**：**只有该格被消行/消列清除时**才计入（教程原话「Burn lines and collect crystals」）。水晶格 = 「必须打通那一行/列」的定点目标 —— **这是关卡模式与竞速模式的唯一玩法差异**：竞速是"消哪条都行"，关卡是"必须消那一条"。
- **难度曲线**（30 关首批，`levels.js` 数据表驱动）：

| 关段 | 引入 | 意图 |
|---|---|---|
| 1-3 | 空盘 + 1 种水晶 ×3，同一行 | 教「消行才收集」 |
| 4-8 | 2 种水晶，分散行列 | 教「要规划先打哪条」 |
| 9-15 | 预置阻塞块 | 教「空间要省」 |
| 16-24 | 3 种水晶 + 水晶在角落/边列 | 边列最难填满 = 真难点 |
| 25-30 | 石块障碍 + 大量预置 | 收尾 |

- **胜利结算**（截图 4）：`Excellent Game!` + 金币 + 星星，`Tap to continue`。
- **失败**：无处可放 → 「重来 / 看广告换一批托盘继续」（§9 的红线内）。

### 6.1 灰色石块（截图 9）

**定义为：不可消除的惰性格** —— 占位，且**一行里有石块 → 那行永远消不掉**。作为关卡障碍从棋盘预置出现。
理由：给关卡设计师一个强力的空间约束工具，且规则只有一条（"石块不消失"），零认知负担。
（截图里它出现在**托盘**，若实机确认是"可放置的惩罚块"，那更狠也更有趣 —— 见 §17。）

### 6.2 三星评级

过关瞬间按**剩余空格数**评：`empty ≥ 24` → ⭐⭐⭐；`≥ 12` → ⭐⭐；否则 ⭐。**本局有 perfect clear → 直接保底 ⭐⭐⭐**。
理由：给纯克隆最缺的**重玩理由**；评价维度是"打得干净"而非"运气好"；星星喂章节宝箱（§10）形成闭环。

## 7. 模式 B：对手竞速（P3）⚠ 这是我们自创，不是抄

**调研更正**：原版 Block Blast **没有**对手进度条/实时对战，只有异步排行榜（截图很可能来自山寨版或网页换皮）。所以这个模式**没有被市场验证过**，我们是在自创机制 —— 要么做出彩，要么果断砍掉，别因为"截图里有"就做。

- **它的设计价值**（值得做的理由）：把「无尽模式的空虚计分」切成**一连串 30 秒兑现的小胜利**。无尽模式的留存痛点就是分数没有中间反馈，这个机制正好补上。
- **实现**：**本地 bot 队列，不需要服务器**。一条递增分数线表（`[300, 600, 900, 1400, 2000, 2800, …]`，末尾按公式外推到无限），每条线配一个头像+名字。超过一个 → 头像换下一个 + 进度条前进 + 「你已击败 N 位玩家」（社会证明弹窗，调研显示这是被验证有效的钩子）。
- 顶部大黄数字 = 待超越的分数线；旗帜 🏁 = 当前得分（截图 5-10 的语义解读）。
- **诚实红线**：bot 名字/头像必须**明显是虚构角色**（不能伪装成真人玩家排行榜）—— 否则是欺骗性设计，也是商店合规风险。
- 无失败罚：卡死即结束，进度条到哪算哪。

## 8. Juice / 反馈（调研里被玩家点名夸的，逐条抄）

> **这一节的优先级 = 核心逻辑**。玩家原话里被夸的全是这些；规则谁都会写，juice 才是产品本体。

| 时刻 | 反馈 |
|---|---|
| 落子 | 下压 + 落地弹（scale 1.05→1.0, 100ms）+ 短促"啪" + 轻震动 |
| 消行 | **发光 → 碎裂 → 拖出彩虹粒子尾迹**（玩家点名的具体画面）；**逐格延迟扩散消失**（每格 20ms，从消除中心向两端）——不是整行同时消失。扩散感 = "我引爆了它" |
| 多条同消 | 屏幕轻震（幅度随 L 递增）+ 中央弹出 **升级式赞美词**：`Good!` → `Great!` → `Amazing!` → `Unbelievable!`（原作的招牌，玩家点名） |
| streak | 音效**音高沿音阶上行**（C-D-E-F-G…，随 streak 递增）—— 最便宜也最上瘾的正反馈；显示 `COMBO ×2.5` |
| **perfect clear** | §4 的全屏白闪 + 金光爆开 + `PERFECT!` + 最高音效 + 长震动 |
| 收集水晶 | 水晶从消失格**飞向顶部目标条**（贝塞尔，300ms）+ 数字跳动 + "叮" |
| 危险 | 当 `fill ≥ 0.75` 时，分数区的**心跳图标开始加速跳动**（原作的招牌反馈，玩家点名）——不给文字警告，只给生理紧张 |

## 9. 变现：三条红线 + 必做的去广告 IAP（§0.1 结论 2 —— 品类第一杀手）

### ⛔ 三条红线（调研里被骂得最狠的，逐条禁止）

1. **绝不局中插屏。** Woodoku 的重灾区：广告盖住棋盘导致误放块——**它直接偷走玩家的一局**，比"广告多"毒得多。
2. **绝不在玩家拒绝复活广告后强塞一个无奖励广告。** 这是 Block Blast 被骂最狠的一条，摧毁的是信任不是耐心：**它拿走了玩家"我不看"的选择权**。
3. **输局后的插屏必须短且低频**（≤15 秒可跳过，且**每 3 局最多一次**）。玩家原话："游戏一局不到 5 分钟，凭什么让我看 3 分钟广告"。

### ✅ 必做

- **一次性「去除所有广告」IAP**（$2.99 档）。Woodoku 被骂多年的正是**没有这个选项**——「我愿意付钱你都不让」比广告本身更招恨。**这也是我们最干净的收入来源**。
- **激励视频**（全部是玩家**主动**选择的，`Ads.rewarded`）：换托盘 / 撤销 / 失败续命（每局限 2 次）/ 领金币。
- 每游戏自备 adUnits，**绝不复用他游戏的**（引擎契约）。

### 道具与金币

| 道具 | 作用 | 阶梯 |
|---|---|---|
| **刷新** | 弃掉托盘 3 块换一组 | **每 8 次落子充能 1 格**（截图 2 灰=没电、截图 3 绿=有电）；有电时可看广告立即用 |
| **撤销** | 撤回上一步（含消行复原、水晶回退、streak 回退） | 本局免费 ×1 → 看视频 → **金币 300**（截图是 1300，那是明显的逼氪价，**我们不装这个坑**） |

金币来源：关卡结算（20-40，按星数）、竞速结算、perfect clear、看广告。用途：买撤销/刷新。
**⚠ 调研警示**：绝不做「看广告赚钱/提现」型换皮（该品类已有多起诈骗投诉），会把整个产品的信任拉低。

## 10. 进度与留存

- **章节地图**：30 关分 3 章，每章末**星星宝箱**（凑够 X 星开）→ 金币 + **皮肤**。三星评级的兑现出口。
- **皮肤**（薄差异化，本仓家风）：方块主题 4 套（糖果/宝石/像素/深海），`PAL` 调色板切换，**渲染函数必须确定性、禁 `Math.random`**（snake 实踩）。
- **每日谜题**（原作有，成本低、留存高）：每天一个固定种子的关卡，全球同一题。
- **竞速最高分** + 累计击败对手数 + **累计 perfect clear 数**（我们的招牌统计）。
- **成就**（薄，~30 条，数据表驱动）：最长 streak、单局 perfect clear ×2、零撤销通关、连续 7 天…
- **只存**：关卡进度/星星/金币/皮肤/成就/最高分/当局快照。**不存永久数值加成**（会毁掉分数可比性）。

## 12. 工程（严守引擎契约，见 `engine/README.md`）

### 12.1 模块分层（index.html 加载顺 = 依赖顺，load-bearing）

引擎脚本 → `prng` → `pieces` → **`dealer`（§2，单独一层：命门 + 重度单测 + 公平性可验证）** → `levels` → `bots` → `core`（棋盘/放置/消除/计分/streak/**perfect clear**/game-over/收集/撤销栈）→ `storage` → `drag` → `fx`（§8 粒子/动画队列）→ `render` → `main`。

- **纯逻辑层**（双导出 `if(module.exports)…else 全局`，node 可单测）：`prng / pieces / dealer / levels / bots / core`。改这些**先写测试**。
- **DOM/渲染层**：`render`（`renderAll()` 契约：每帧 `clearHits()` 全屏重画 + `addHit()`）、`drag`、`fx`、`main`。
- **状态**：单一可变 `G`（`G.phase`：menu / playing / dragging / win / lose）；所有交互进 `dispatch(action, data)`。
- **撤销栈**：只存 1 步（落子前的棋盘/分数/streak/收集/托盘快照），避免存档膨胀。
- **零硬编码文案**：`T('blockblast.xxx')` + `locales/<lang>.json`（**必须嵌套结构**——扁平写法查不到且零报错，snake 实踩；`check-locales` 查不出这个）。en 基准 + zh-CN。canvas 非定长文案过 `wrapLines`。
- **存档**（`CFG.key('run')`，带 `v: N`）：进度/星星/金币/道具/皮肤 + 当局快照（棋盘/托盘/**下一组**/分数/streak/目标/DDA 状态/PRNG 种子）。**改 `G` 形状必 bump `SAVE_VERSION`（旧档丢弃不迁移）；改任何 js/css 必 bump index.html 所有 `?v=N`。**

### 12.2 测试

- **node 单测**：放置合法性 / 消行消列 / **计分公式与 streak 边界**（断裂需连续 2 回合零消除、×4 封顶）/ **perfect clear 判定与奖励** / **game-over = 且仅当三块都放不下** / **`dealer` 硬保证：补块后至少一块可放**（10k 随机棋盘）/ **`dealer` 公平性：传入分数与不传输出逐字节一致**（§2.1 第 2 条）/ **`dealer` 无历史记忆**（§2.1 第 3 条）/ 水晶只在消除时计入 / 撤销精确复原 / PRNG 可复现。
- **auto-player 冒烟**：贪心 AI 跑 1000 局 → 不崩、无 NaN、分数单调不减，并**统计「三块全可放回合占比」落在 60-80%**（手感回归测试）。
- **E2E（playwright 无头）**：拖拽落子（pointer 序列）、消行、perfect clear、过关三星、道具三态、存档续玩、语言切换。
