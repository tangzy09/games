> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 接上广告钩子——**死亡复活**（最高价值位）、**看广告换金币**（喂金币经济）、**局间插屏**，外加 GDPR 隐私选项按钮。

**⚠ 先说清楚现状**：本阶段接的是**代码钩子**。**真实广告收入要等**：① iOS 壳 + AdMob 真广告位（见 `~/.claude/skills/admob-monetization`），或 ② 铺游戏门户（`html5-game-portals`）。现在 web 端 `Ads` 会自动降级成 `confirm()` 模拟——**能测，但没收入**。钩子必须先接好，否则上架/铺门户时来不及。

**Architecture:** 复活的盘面操作是**纯逻辑**（进 `tools.js`，可单测）；广告调用与相位流转在 `main.js`；按钮在 `render.js`。引擎 `Ads` 已在 `boot()` 里 `init()` 过，直接用：

- `Ads.showInterstitial()` → `Promise<boolean>`
- `Ads.showPrivacyOptions()` → `Promise<boolean>`（原生才有；GDPR 要求可随时撤回同意）

**必读**：`engine/ads.js`（真实 API）、`games/abyssshoot/CLAUDE.md`、`games/snake/js/main.js`（`REVIVE`/`Ads.showRewarded().then(...)` 的既有写法）。

## 三个投放位的设计

| 位置 | 触发 | 给什么 | 限制 |
|---|---|---|---|
| **☠️ 死亡复活** | DEAD 画面的「📺 继续」按钮 | **每列削掉顶部 3 格** + `dead=false` + 继续玩 | **每局限 2 次** |
| **🪙 换金币** | 道具栏旁的「📺 +100」按钮 | **+100 金币** | 无（看广告本身就是成本） |
| **📺 插屏** | RESTART 时（**不在死亡那一刻**，别打断情绪） | — | **每 3 局一次** |

**⚠ 复活为什么削「顶部」而不是「底部」**：两者同样降低列高（同样远离死线），但**顶部是最老的杂鱼、底部是你辛苦垒起来的大鱼**。削底部等于**毁掉玩家的成果**；削顶部保住它。这是个反直觉但正确的选择。

```javascript
// ☠️ 看广告复活:削掉每列**顶部**的 n 格,清除死亡,继续玩。
// ⚠ 为什么削顶部而不是底部:两者同样降低列高(同样远离死线),但**顶部是最老的杂鱼、
//    底部是玩家辛苦垒起来的大鱼**。削底部 = 毁掉玩家的成果;削顶部保住它。
//    (棋盘 index0 = 顶 = 远离玩家;末尾 = 底 = 玩家侧/死线。)
// ⚠ 削完必须 resolve:移除会制造新的相邻,可能触发连锁。
function revive(s, n) {
  s.events = [];
  const k = Math.max(1, n || REVIVE_ROWS);
  for (let c = 0; c < s.cols; c++) s.board[c] = s.board[c].slice(k);   // 砍掉顶部 k 格
  s.dead = false;
  s.events.push({ t: 'revive', rows: k });
  CORE_T_.resolve(s);
  return { ok: true };
}
```

在「道具」用例之后插入：

```javascript
  // ── 广告:看广告换金币 ──
  await page.evaluate(() => { dispatch('RESTART', {}); G.noAnim = true;
                              for (let i = 0; i < 4; i++) dispatch('SHOOT', { col: i % 5 });
                              G.save.coins = 0; });
  await page.evaluate(() => dispatch('AD_COINS', {}));
  await page.waitForFunction(() => window.G.adBusy === false, { timeout: 8000 });
  const adc = await page.evaluate(() => G.save.coins);
  if (adc !== 100) throw new Error(`看广告应 +100 币,实为 ${adc}`);
  console.log('OK 看广告换金币:+100');

  // ── 广告:死亡复活(削顶部,列高应下降) ──
  await page.evaluate(() => {
    dispatch('RESTART', {}); G.noAnim = true;
    // 造一个必死盘:一列填满不相邻同值
    const alt = []; for (let i = 0; i < G.s.rows; i++) alt.push(i % 2 === 0 ? 2 : 4);
    G.s.board = [alt, [], [], [], []];
    G.s.ammo = 8;
    dispatch('SHOOT', { col: 0 });
  });
  await page.waitForFunction(() => window.G.phase === 'DEAD', { timeout: 5000 });
  const d0 = await page.evaluate(() => ({ h: G.s.board[0].length, revives: G.revives }));
  await page.screenshot({ path: path.join(SHOT_DIR, 'e2e-ad-revive.png') });
  await page.evaluate(() => dispatch('REVIVE', {}));
  await page.waitForFunction(() => window.G.adBusy === false && window.G.anim === null, { timeout: 8000 });
  const d1 = await page.evaluate(() => ({ phase: G.phase, h: G.s.board[0].length, revives: G.revives, dead: G.s.dead }));
  if (d1.phase !== 'PLAYING') throw new Error(`复活后应回到 PLAYING,实为 ${d1.phase}`);
  if (d1.dead) throw new Error('复活后 core 的 dead 应清除');
  if (!(d1.h < d0.h)) throw new Error(`复活应削掉格子(列高 ${d0.h}→${d1.h})`);
  if (d1.revives !== 1) throw new Error('复活次数应 +1');
  console.log(`OK 死亡复活:列高 ${d0.h}→${d1.h},回到 PLAYING`);

  // ── 复活次数用尽后,按钮不再生效 ──
  await page.evaluate(() => { G.revives = 2; G.phase = 'DEAD'; dispatch('REVIVE', {}); });
  const spent = await page.evaluate(() => G.phase);
  if (spent !== 'DEAD') throw new Error('复活次数用尽后不该还能复活');
  console.log('OK 复活上限:用尽后点不动');
```
