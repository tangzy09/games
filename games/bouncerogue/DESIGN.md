# Bounce Rogue / 弹球肉鸽 —— 设计文档

> 工作名，可爱风，名字/美术二期定。**经典 Breakout × 肉鸽**，融本仓「经典游戏×轻收集」家风。
> 玩法骨架**抄 Roguebreaker**（已上架验证的正统 Breakout-roguelite，Steam 731420，闭源——**只抄机制/数值，代码从 straker 的 250 行 canvas 底子自写，美术/名称/文案全原创**，同 mines 抄 Dragonsweeper 的老规矩），
> build 深度借 **BALL x PIT** 的合球系统，收集/图鉴/皮肤/成就沿用本仓 snake 的做法。
> root `CLAUDE.md`（引擎契约、部署铁律、git 纪律、iOS 流水线）先读，本文件只讲本游戏专属设计。

## 0. 一局是什么样（一句话）

底部横板顶球向上砸砖，**碎够砖就二选一升级**，**分越高球越快**，一命到底、场上球全漏光掉一命、命光从头来。关卡从预设池**无尽随机**抽，越滚越强也越快。差异化 = 可爱收集图鉴 + 球种养成/合球 + 一种会下爬的「下降砖」当威胁点缀。

## 1. 核心循环（抄 Roguebreaker）

1. 一面砖墙，横板顶球碎砖（经典 Breakout 物理：AABB 碰撞、板反弹角度按击中位置偏转）。
2. **碎够 N 块砖 → 弹二选一升级**（不是清完关才给；用「碎砖攒槽、满则弹」实现，短暂子弹时不全停）。
3. 清完当前关的砖 → 从**预设关池随机抽**下一关（心里有数、但不确定是哪张）。
4. **分数→球速**：分越高全场球越快（Roguebreaker 招牌难度爬升）；**多球是双刃剑**——球多更爽但更快更乱。
5. **一命到底 permadeath**：场上球**全部漏光** = 掉一命 + 重发一球（对应用户定的「多球池、全掉才重置」）；**命光 = run 结束、从头来**。
6. 通过里程碑（累计关数/分数）→ **下一 run 换一批升级选项 / 解锁新卡进池**（Roguebreaker 的 run 间新鲜感）。

**唯一失败条件**：命归零（学 mines「一条干净规则」）。命 = 初始 3~5 条，掉球全漏光扣 1。

## 2. 抽卡经济（防决策疲劳）

- **二选一**（比三选一清爽，抄 Roguebreaker），带**重抽 / banish（划掉不想再见的卡）**。
- 卡分两类：**可叠数值卡**（反复抽、叠层：+球速控制/+暴击/+碎砖攒槽速率/+分）与**唯一遗物卡**（拿过不再出：双板、激光、幽灵球、连锁闪电、慢动作、重力倾斜、磁吸召回…）。
- 稀有度 普通/稀有/史诗。卡与卡有 synergy（火球+穿透、分裂+暴击、磁吸+激光）。
- 升级技能样例（抄 Roguebreaker）：慢动作、球吸回板（召回瞄准）、发射导弹（板激光）、倾斜板面（重力偏球）、加发一球。

## 3. 球种养成 + 合球（borrow BALL x PIT，本游戏的 build 深度 & 收集核心）

- 球本身是**收集/成长单位**：不同球种（普通/火球爆炸/穿透/闪电/分裂/追踪…）。
- **同种球叠到 3 级 → 进化**成配方指定的新球（**进化配方图鉴**，收集钩子直接长在图鉴上）；**两颗无配方的 3 级球 → 融合**，新球叠加双方效果。
- **场上球种有上限（如 4 种）** → 逼玩家取舍、凑配方，而不是无脑堆。
- 与第 2 节的卡是两条正交轴：**卡 = 全局 buff / 板与规则**，**球种 = 攻击载体养成**。

## 4. 砖块生态（可图鉴，初版 ~8 种，先少后加，学 mines 怪物表）

普通(1击) · 硬砖(多击带血条) · 爆炸砖(连锁清邻) · 掉落砖(碎了掉即时道具) · 硬石(不可碎，只能被穿透/爆炸/导弹处理) · **下降砖**(会逐拍往下爬，爬到板线=直接扣命，我原「下压传送带」核心**降级成这一种砖**当威胁点缀) · 减速砖(碎了短冻球速) · **Boss 砖**(每 X 关出，大血，碎了清一大片=高潮，掉专属遗物/进化球)。

## 5. 道具（掉落砖里掉，抄 Roguebreaker）

多球 · 加速/减速 · 加分 · 攒槽加速（"加花样"）· 短护盾（底部接球网 N 次）。**双刃**：多球/加速让局面更爽也更险。

## 6. 元进度（轻收集，不给永久数值加成——用户定，规避 mines「堆升级系统失乐趣」的坑）

- **图鉴**：遇过的**升级卡 + 球种 + 合球配方 + 砖种**全记录（DOM 浮层，学 mines codex + snake gallery）。
- **里程碑解锁**：累计关数/碎砖数 → 解锁新卡/新球种进池 + **皮肤**（板/球/砖配色，学 snake 4 主题，`render` 的 `PAL` 可切换引用，遮罩/纹理函数必须确定性、禁 `Math.random`）。
- **成就墙 ~120**（学 snake：阶梯累计族 + 单局族，数据表驱动 + 运行时断言，别写死判定）。
- **只存**：图鉴/解锁/成就/最高分/皮肤/当局快照。**不存**永久数值。

## 7. 广告钩子（接引擎 `Ads`，学 snake；每游戏自备 adUnits，绝不复用他游戏）

- **激励-复活**：命光时看广告 → 满命续 run（每局限 2 次）。
- **激励-加卡/重抽**：二选一时看广告出第 3 张或重抽。
- **插屏**：每清 N 关的关间隙弹（不打断关内），AI 局不弹。
- **AI 代打**：二期候选，不进初版；若做则照搬 snake 反刷规则（AI 局减半分/不刷纪录/单独计）。

## 8. 工程（严守本仓引擎契约，见 `engine/README.md`）

- **模块分层**（index.html 加载顺 = 依赖顺，load-bearing）：
  引擎脚本 → `prng` → `bricks`(砖种数据) → `balls`(球种/合球配方数据) → `core`(物理/碰撞/关卡/球速/命/攒槽) → `cards`(卡池+施法) → `codex` → `storage` → `achievements` → `themes` → `render` → `main`。
- **纯逻辑层**（双导出 `if(module.exports)...else 全局`，node 可单测）：`prng/bricks/balls/core/cards/achievements/themes` + codex 数据函数。**改这些先写/跑测试**。
- **DOM/渲染层**（浏览器专用，E2E + 无头截图验）：`render`(renderAll 契约、offscreen 双层)、`main`(boot/主循环/事件消费/存档/面板)、codex/皮肤 DOM。
- **游戏时钟非墙钟**：球速爬升、下降砖节拍、慢动作/冻结时长、攒槽窗口全走 `gameMs`（每 tick `+= interval`）；暂停/切后台冻结。**core 禁 `Date.now()`**（唯一豁免：换种子快照）。
- **core 事件流 `s.events`**：每 tick 清空重填类型化事件（brickBreak/combo/charge/cardOffer/ballLost/lifeLost/levelClear/descentHit/bossSpawn/fuse/death…），成就引擎与音效统一消费，替代散落 flag（学 snake）。
- **PRNG 播种**：关卡生成 + 卡池抽取 + 掉落，全走 `PRNG`（mulberry32），可复现、可测。
- **存档**（storage.js）：单 JSON blob（snake 风）= 元进度(图鉴/解锁/成就/最高分/皮肤) + 当局快照(砖网格/球群含球种等级/已选卡/关号/命/分/gameMs) 支持中途关页续玩。**`SAVE_VERSION` 纪律**：改 G/存档形状必 bump，旧档丢弃不迁移；改 JS 必 bump index.html 里 `?v=N`。保守合并的**开放 map 字段（如 `specials{}`/动态计数）必须保持空默认**，否则每次 load 清空动态 key（snake 实踩的 Critical）。
- **零硬编码文案**：全走 `T('dotted.key')` + `locales/<lang>.json`（**必须嵌套结构**，扁平写法查不到且零报错——snake 实踩），en 基准 + zh-CN，商店页十语。`node tools/check-locales.js` 必 0 fail。canvas 非定长文案过 `wrapLines`。
- **iOS 壳**：自备 `package.json`(Cap6，抄 minesweeper)、`capacitor.config.json`(appId/appName)、`resources/`(icon 1024 + splash 2732)；`codemagic.yaml` 加一段 workflow 填 4 var（GAME_DIR/BUNDLE_ID/APP_STORE_APP_ID/GAD_APP_ID）。
- 从 `games/_demo` 起步，新目录 `games/bouncerogue/`。

## 9. 测试策略 + 两条边界红线（⚠ 诚实修正，别让后人误抄 snake 的保证）

- **⚠ 打砖块没有 snake 那种「保证零死亡」的 AI**：snake 靠哈密顿回路数学闭合才能保证；打砖块的板是**反应式**的，物理上**无法保证**接住所有球或及时清关。故：
  - ① core 物理/碰撞/球速/攒槽/卡牌施法/合球配方 → **node 单测**（浮点在同一 JS 引擎内确定、PRNG 复现）；
  - ② auto-player 只做**冒烟测试**（跑 M 关，断言不崩溃 / 无 NaN / 不变量成立<球数≤上限、命≥0、砖网格形状> / 卡效与合球正确），**不断言零死亡**。
  - ③ E2E（playwright 无头）：整局流程 + 二选一 + 存档续玩 + 皮肤 + 图鉴 + 广告钩子跑通。
- **⚠ 下降砖 vs 连续球的碰撞边界**：下降砖降格那一刻若与球体重叠，须对球做**推出**（或球贴砖面时该砖本拍不降），防包球穿模——真机偶现、全新档 E2E 测不出，必须显式处理 + 单测覆盖。
- **⚠ 合球触发边界**：进化/融合发生在 tick 内、可能连锁（融合出的球又满足新配方）——须定「每 tick 最多结算一次合球 / 或迭代到稳定并防死循环」，写进单测。

## 11. 分阶段实施（学 mines 教训：先把核心循环做好玩，别一上来堆系统）

1. **P1 核爽**：`core`（Breakout 物理 + 碰撞 + 命/多球池全漏扣命 + 分数→球速 + 碎砖攒槽二选一 + 无尽随机关）+ 最简板球 + 5 张卡 + node 单测 + 无头冒烟 → **先确认「碎砖攒卡、球越多越险」这套手感成立**再往下。
2. **P2 build**：卡池扩到 ~40（叠加/遗物/重抽 banish）+ **球种养成 + 合球（进化/融合/球种上限）** + 砖种 8（含下降砖/Boss）+ 事件流接音效。
3. **P3 元层**：图鉴（卡/球/配方/砖）+ 里程碑解锁 + 皮肤 + ~120 成就 + 存档续玩。
4. **P4 变现/上架**：Ads 钩子 + i18n 补 + iOS 壳 + 商店素材。AI 代打若做放最后。
