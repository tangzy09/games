## 这个游戏的唯一卖点：**每一局都存在解法，而且我们证明给你看**

⛔ **但措辞必须精确到变态 —— 这里埋着一颗语义炸弹（红队抓出，DESIGN §2.1）**：

论文的 81.9% 可解率是 **thoughtful solitaire（透视全部 52 张牌）** 下算的。**玩家看不见暗牌。**

| ❌ 绝不能说 | ✅ 只能这么说 |
|---|---|
| 「这局你一定能赢」 | 「**本局存在解法**」（已由求解器证明） |
| 「是你还没找到」 | 「这局从第 12 步之后就没有解了」（**陈述事实，不指责**） |

**为什么这是死线**：玩家在第 12 步时，**在他能看到的信息下根本无法避免那一步**。指责一个**信息上不可避免**的错误，会造出比「你坑我」更毒的差评：**「它说是我走错，可我根本没法知道」** —— 而且是我们主动挑起的。

⭐ **反过来，主动公开这个落差就是护城河**（没有竞品敢写）：

| 模式 | 理论可解（透视） | **盲打 AI**（看不见暗牌，和玩家一样） | 真人（调研） |
|---|---|---|---|
| draw-3 | 81.9% | **7.6%** | ~11% |
| draw-1 | 90.5% | **32.3%** | 30-40% |

⇒ 公平页要写：**「这局有解」和「你能赢」之间隔着 58-74 个百分点。我们能保证的是这局不是死局，不能保证你一定赢。**

## `tools/sim-blind.js`（盲打 AI）是本作最重要的工具

它**不透视暗牌**，只用玩家看得见的信息决策。三个用途：

1. **验证规则实现是对的** —— 规则写错的 Klondike，AI 是赢不了 7.6% 的局的（上表两个模式都落在真人区间内 = 最强验证）
2. **量化「有解」vs「人能赢」的落差**（公平页的数字）
3. **难度分档 + 回归基线**（改了 `deal`/`rules` 之后不劣化）

⚠ 第一版启发式胜率 **0%** —— AI 在两列之间来回搬牌。**不翻暗牌、不清空列的 `tt` 搬运几乎总是坏棋**（把牌埋起来 + 浪费步数）。修正后才对上真人区间。

```bash
npm run sim:sol 500 3     # draw-3 盲打胜率（基线 ~7.6%）
npm run sim:sol 800 1     # draw-1（基线 ~32.3%）
```

## 几条不能动的地基

- **`deal.js` 必须是可复现的纯函数**（`deal(seed) → 完全相同的牌局`）。存档存 `seed + move list`、牌局编号、可解 seed 池**全靠它**。⛔ **绝不允许 `Math.random` 进那个文件**（单测钉死）。
- **撤销 = 重放 move list 到 n−1**，不是快照栈 ⇒ 存档几百字节；且天然没有「恢复后不能撤销」这个品类经典一星。它**也正是将来验证 solver 解的机制**。
- **draw-3 的回收保序** —— 这一条**直接改变可解率**，必须与离线 solver 的模型一致，否则「已验证可解」是系统性谎言（DESIGN §2.4）。
- **draw-1 / draw-3 是两个不同的可解性问题**（同一个 seed 在 draw-3 无解、draw-1 可解是常态）⇒ 池要分片；**draw 模式是开局前属性，局中不可改**。

## ⚠ autoplay 的安全规则有个真 bug（已修，别改回去）

经典写法「一张牌可无脑收 ⟺ 两个异色 foundation 都到 `rank−1`」**在 rank ≤ 2 时过于保守**：
黑 2 收走后**没有任何牌需要它承接**（红 A 直接进 foundation，不需要落在黑 2 上）。
⇒ **A / 2 无条件安全**。不修的话玩家会看着一张明明该收的 2 不被收走，觉得「它怎么不收」。

## 变现红线（被差评逼出来的，一条都不能破）

1. **撤销 / 提示 / 重开 / 换一局 —— 永远免费，永远不看广告。** 这是纸牌的基本人权。
   玩家原话：「他们**故意让你更容易无路可走**，好逼你看广告拿提示。」
2. **绝不局间连播插屏**（微软的「12 连播」是这个品类最致命的叙事）。
3. **不要体力，不要押注式金币**（Klondike 玩家零容忍）。
4. **一次性去广告 IAP，不是订阅**（「付费还看广告」是微软最毒的一条差评）。
5. **横幅绝不遮牌** —— 布局为它**预留**空间（`L.bannerH`），不是盖上去。

**主力收入是横幅**：纸牌单次会话 10-15 分钟（所有品类里最长的之一）⇒ 横幅曝光时长极高且不打断。
激励视频只用于**纯增益**（牌背/桌布皮肤），且必须有**消耗端**（收集系统），否则那条腿约等于零收入。

## 工程要点

- **纸牌瀑布**（产品的心脏，不是彩蛋）：累积拖尾与引擎「每帧全屏重画」的契约**正面冲突**
  ⇒ **持久 trail canvas（永不清）+ 主 canvas 每帧只画运动中的牌**。这是本作在引擎契约上打的洞。
- **牌面必须 sprite 缓存**：52 张牌每帧重画 = 每帧几百次 `fillText` + 圆角路径，不缓存必掉帧。
- **hit 区按底→顶注册**：engine 的 `hitTest` 是**后注册优先**（倒序遍历）。反过来写，点一叠牌会命中**最底下那张** —— 症状诡异的必然 bug。
- **tap-to-move 和拖拽都要支持**：65+ 用户手抖/关节炎**拖不准**，不给点击移动他们根本玩不了（这不是矫情）。tap/drag 仲裁：位移 <10px 且时长 <250ms = tap。
- **四色牌**（♠黑 ♥红 ♣绿 ♦蓝）：无障碍标配，对老花/色觉衰退用户**价值高于大字号**。
- **统计双口径**：总胜率 / **零撤销·零提示胜率**（后者才是玩家会拿去炫的）。无限撤销会把胜率架空 —— 不定口径，统计就是假的。
