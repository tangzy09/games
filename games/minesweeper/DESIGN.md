## 1. 为什么不是 spam

### 1.1 竞品分析

| 品类 | 克隆困境 | 我们的解 |
|---|---|---|
| **Minesweeper** | App Store 已有 50+ 克隆，全是 emoji + 广告轰炸 | ✅ **天使图收集**（独占）+ **无广告骚扰** |
| **vs Snake** | 同家公司的完全不同玩法 | ✅ **系列品牌**（官网可互推） |
| **vs Solitaire** | 逻辑游戏的竞争维度不同 | ✅ **竞速排行 vs 思考纸牌** —— 用户群体不重叠 |

### 1.2 商店文案策略（避免克隆标签）

- ❌ 不说："Minesweeper Clone" / "Classic Minesweeper"
- ✅ 改说："Minesweeper Angel" / "Logic Puzzle Collection"
- ✅ Keywords：`minesweeper` (公有品类，合规) + `logic puzzle` + `angel collection` + `casual game`
- ✅ 品类归类：**Puzzle** 而非 Board（后者更容易被审视为克隆）

### 1.3 产品质感差异（专业化）

| 维度 | 克隆应用 | 我们的版本 |
|---|---|---|
| **美术** | emoji 或扁平图标 | ✅ 500 张精致天使插画 + 主题皮肤 |
| **加载速度** | 秒启 + 10 个广告 | ✅ 秒启 + **绝无局间连播** |
| **玩法深度** | 单盘无尽循环 | ✅ **100 关 + 500 张天使 + 排行 + 成就 120** |
| **国际化** | 英文 only | ✅ **10 语 UI + 39 语 ASO** |

## 2. 核心玩法（保留经典，零创新风险）

### 2.1 难度档位

```
初级 (8×8, 10 雷)      — 新手友好，5-10 秒通关
标准 (9×9, 10 雷)      — 经典 Windows，20-40 秒通关（最重要档位）
中级 (12×12, 30 雷)    — 进阶，1-3 分钟
高级 (16×16, 40 雷)    — 挑战，3-10 分钟
传奇 (16×30, 99 雷)    — 终极，10-20 分钟
```

**为什么这个分级**：
- 初级 → 保证新手能赢（迅速获得天使图，建立自信）
- 标准 → 经典难度，玩家群体最大
- 中级-高级 → 渐进式挑战，满足进阶玩家
- 传奇 → 长尾用户竞争（排行榜的天花板）

### 2.2 基础规则（纯 Windows 扫雷）

- **左键**：打开格子（若为空白，递归打开周围 8 格）
- **右键/长按**：标记雷（防误点，第三次解标）
- **数字**：周围 8 格的雷数
- **胜利条件**：全部格子打开 OR 仅剩雷未打开
- **失败条件**：点到雷

**不改机制**的原因：
1. 规则已验证好玩，无需创新风险
2. 玩家预期明确（无surprise mechanic = 信任）
3. 降低测试成本（core.js 单测已有参考）

### 2.3 唯一的「小优化」——长按转向（操作爽感）

```
长按 1.5s：进入「标记模式」
  ├ 绿色边框围住当前格
  ├ 再点击 = 标记 / 解标
  ├ 点其他格或点确定 = 退出模式

好处：
  · 防误点（特别是高龄用户 + iPad）
  · 手感接近 Snake 的转向缓冲
  · 提升「精度星」的获胜概率
```

**E2E 测试必验**：长按转向不能破坏快速点击（竞速玩家要求）。

## 3. 天使图收集机制（核心 → 长期参与）

### 3.1 映射关系（算法设计）

**前 100 张**（按难度阶梯分配，**保证所有玩家能 100% 解锁**）：
```
初级 (8×8)   →  天使图 #1-20     （每 5 张不同主题：樱花/海洋/星空/…）
标准 (9×9)   →  天使图 #21-60    （每 10 张一个系列）
中级 (12×12) →  天使图 #61-80    （难度上升，图片更精细）
高级 (16×16) →  天使图 #81-100   （高端玩家 first clear）
```

**后 400 张**（条件化解锁，驱动**深度参与**）：
```
特殊挑战系列（100 张）：
  ├ 每日盘 7 天连胜 → 解锁 #101-107（每周一张，限时）
  ├ 无错标 20 局 → #108-127
  ├ 速通（标准盘 <15s） × 10 → #128-137
  ├ 周排行 Top 10 × 4 周 → #138-147
  └ …… 共 100 张

排行赛系列（100 张）：
  └ 每月一个主题赛，前 100 名分别解锁 #201-300

收藏家系列（100 张）：
  └ 集齐某难度全★3 → 解锁该难度的「完全体」天使（10 张）

季赛系列（100 张）：
  └ 每季四个主题月赛
```

**设计亮点**：
1. 前 100 张**零挫折路线** —— 任何玩家都能到达
2. 后 400 张**多条路线** —— 选择参与（速度/精度/收集/排行/季赛）
3. **路线不排斥** —— 同一张天使可能通过多种条件解锁，但只计一次

### 3.2 进度显示（主界面）

```
┌─────────────────────────────────┐
│  🎁 已集齐 237/500 天使图       │ ← 大进度条（居中，醒目）
│                                 │
│  ⭐ 下一个系列：「无错标 20 局」 │ ← 当前进度路线
│     ███░░░░░░░░ 8/20            │
│                                 │
│  🏆 月排行名次：#47 / 1,234     │ ← 实时排行
└─────────────────────────────────┘
```

### 3.3 图鉴界面（收集品展示）

```
分类标签：【全部】【樱花系】【排行赛】【成就】…

缩略图网格：
  ├ 已解锁：大图 + 天使名字 + 获得日期
  ├ 未解锁：灰黑色 + 「差 X 次」提示（指向最近的解锁路线）
  └ 可导出：图鉴页面顶部 🖼️ 按钮 → 选天使 → 导壁纸
```

### 6.2 爽感动效（关键差异）

```

  └ 波纹递归扩散（0.3s per wave），不是瞬间秒开

胜利界面：
  ├ 棋盘缩放回弹（1.2× → 1.0）
  ├ 粒子爆发（中心向外）
  ├ 星光闪烁 + 配音「胜利」
  └ 延迟 0.8s 显示结算浮层

踩雷：
  ├ 雷位置闪红 + 抖动
  ├ 全盘其他雷缓慢显现（淡入，1s）
  └ 配音「失败」（可关闭）

时间压力提示（传奇盘）：
  └ 倒计时最后 10s 时，数字变红 + 脉动
```

**性能保证**：
- 所有动效走墙钟（不进 core，不存档）
- 可通过「无障碍 - 减弱动效」完全关闭（`prefers-reduced-motion`）

## 7. 变现策略（零破坏游戏体验）

### 7.1 广告策略

**横幅广告**（主力）：
```
位置：棋盘下方（预留 L.bannerH = 60px）
时机：每局间隙显示，非局中
密度：单局 5-15 分钟 → 高曝光，低骚扰
```

**激励视频**（增值，非强制）：
```
触发点：
  ├ 「再来一局」时可选看广告 → +1 额外天使图
  ├ 周末赛前可看 → +100 金币（兑换皮肤）
  └ 新主题解锁时可看 → 免费立即解锁（vs 集齐 N 张）

次数限制：每天最多 5 次（防成瘾）
```

**差评风险红线**（绝对不碰）：
```
❌ 局间连播插屏（微软纸牌的最大诟病）
❌ 强制看广告才能继续玩
❌ 广告干扰棋盘（横幅必须预留空间）
❌ 绕过静音播放（检测系统音量设置）
```

### 7.2 IAP

```
一次性去广告（¥18 / $4.99）：
  ├ 移除所有横幅
  ├ 激励视频改为直接奖励（无视频）
  ├ 有「还原」选项（防锁定购买错误）
  └ 不绑定订阅（一次性，永久有效）

绝不上线的：
  ❌ 体力系统
  ❌ 付费通关提示
  ❌ 付费加速排行
  ❌ 订阅制去广告
```

## 9. 工程路线（4 周上线）

### Phase 1：核心（1.5 周）

```
P1a（盘面生成 + 基础逻辑）
  ├ core.js：5 难度盘生成 + 打开/标记/胜负判定
  ├ 单测：全覆盖（初 / 标 / 中 / 高 / 传奇各 100 次）
  └ 性能：init <50ms，step <10ms

P1b（日常系统）
  ├ storage.js：天使图进度、排行、成就存本地
  ├ dailySeed.js：每天固定种子生成
  └ 连胜判定（夏令时安全）
```

### Phase 2：UI + 收集（1 周）

```
P2a（界面框架）
  ├ render.js：棋盘绘制 + 动效（开格 / 胜利 / 踩雷）
  ├ 主界面：进度条 + Play / 继续 / 图鉴 / 成就 / 皮肤 入口
  └ 图鉴面板：网格 + 筛选 + 进度提示

P2b（成就 + 排行）
  ├ achievements.js：120 条规则数据驱动
  ├ 排行计算（日/周/月）+ 本地排名预估
  └ 排行面板：Rank Card + 对比
```

### Phase 3：美术 + 主题（1 周）

```
P3a（格子美术）
  ├ 5 个主题的样式表（棋盘颜色、数字、旗子、边框）
  ├ Sprite 缓存：52 种组合（5 主题 × 10 状态 + 旗子）
  └ 动效代码（粒子 + 缩放 + 震屏）

P3b（爽感）
  ├ 波纹递归扩散算法（可配 speed）
  ├ 胜利庆祝序列（缩放 + 粒子 + 音效）
  └ 无障碍：减弱动效开关
```

### Phase 4：国际化 + 上架（0.5 周）

```
P4a（i18n）
  ├ locales/<lang>.json：10 语（zh-CN/en/es/hi/bn/pt-BR/ru/ja/pa/de）
  ├ 检查：node tools/check-locales.js games/minesweeper/locales
  └ 商店页 ASO：39 语翻译（AppStore Listing skill）

P4b（E2E + 上架）
  ├ Playwright：通关全流程（成就 + 存档 + 排行）
  ├ iOS 打包（Capacitor + Codemagic）
  ├ ASC 上传（截图 + 视频 + 描述）
  └ 提交审核
```

### 并行可做

- P2b 和 P3 可同步
- 后端排行可先用本地预估，上线后加 EC2（可选 P5）

```javascript
// 每日排行（前 100）
function getDailyRanking(limit = 100) {
  const today = new Date().toDateString();
  const dailyTimes = getAllLocalRuns()
    .filter(r => r.date === today && r.difficulty === 'standard')
    .sort((a, b) => a.elapsedMs - b.elapsedMs)
    .slice(0, limit);
  
  return dailyTimes.map((t, idx) => ({
    rank: idx + 1,
    name: t.playerName,
    time: t.elapsedMs,
    mistakes: t.mistakes,
  }));
}

// 用户排名估算（本地）
function estimateUserRank(myTime) {
  const faster = ALL_RUNS.filter(r => r.time < myTime).length;
  return faster + 1;
}
```
