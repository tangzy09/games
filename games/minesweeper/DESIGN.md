# 扫雷 × 肉鸽 设计文档（v1）

> 2026-07-10 · 代号 `mines`（正式名与美术主题后定）
> 结论先行：**用 Dragonsweeper 的「血量=资源」底座 + DemonCrawl 的 run 结构（关卡链/商店/遗物）**，跑在我们引擎上，免费 web + 十语。

## 一、竞品各自的优势（调研 2026-07）

### 1. 经典竞技流：minesweeper.online / 247minesweeper
- **无猜模式**是现代扫雷的公平底线：盘面 100% 逻辑可解，输=你推理错，不是盘面逼你抛硬币。
- **每日挑战 + 排行榜**：单次尝试上榜，天然回访钩子。
- minesweeper.online 甚至已经加了装备/天赋/任务/竞技场——证明扫雷玩家吃「成长系统」这一套。
- 启示：竞技留存 = 公平 + 每日 + 榜。

### 2. Dragonsweeper（2025 免费 web 爆款，我们最重要的参照）
- **轻改造铁律：只加怪物战斗+升级，不破坏扫雷核心逻辑。**
- 数字语义改造：格子数字 = 相邻**怪物强度总和**（不再是雷的个数）——信息更密，且怪物强度可分级（雷只有 0/1），这是肉鸽化的最佳底座。
- **把「危险」变成可计算的资源**：有 HP，踩怪=扣其强度的血。于是「主动踩怪换 XP/开图」成为策略——扫雷从纯避险变成风险投资。
- monsternomicon（怪物图鉴）：学习怪物行为本身是进度感。
- 单局 5–15 分钟，正对门户时长。
- **关键洞察：血量制本身就化解了传统扫雷最大痛点（50/50 死于运气）**——猜错≠死，只是亏血。所以 v1 不需要昂贵的无猜求解器，肉鸽化自带公平性。

### 3. DemonCrawl（Steam $15，深度天花板参照）
- 完整 run 结构：60+ 关卡链、特殊关（商店/铁匠/公会/灯塔）、无尽模式。
- 物品五类（被动/消耗/魔法/诅咒/传说）1000+；24 种 Mastery（开局流派）。
- 启示：**层间三选一 + 商店 + 遗物池 + 流派**是被验证的深度来源；「诅咒类物品」（负面换爆发）是好设计。

### 我们的空位
Dragonsweeper 是单张大盘一次性玩法（无 run 无成长池无 meta）；DemonCrawl 在 Steam 收费且重度。**「Dragonsweeper 的爽 + 真正的 run 结构 + 免费 web + 十语」没人占。**

## 二、核心设计

### 玩法底座（改造后的扫雷规则）
- 网格盘面，格子内容：空地 / 怪物（强度 1–13 分级）/ 金币 / 道具 / 楼梯（本层出口）/ Boss。
- **数字 = 相邻 8 格怪物强度总和**（Dragonsweeper 式）。
- 点击翻格：空地→安全开图（0 连锁展开）；怪物→受到=强度的伤害并杀死它，掉金币/XP。
- **踩怪是主动策略**：血够时踩小怪开视野、赚经济；算不准时才是赌。
- 插旗改为**标记怪物强度**（长按/模式键），纯辅助记忆。
- HP 归零 → run 结束（激励视频可复活一次）。

### Run 结构（一局 8–15 分钟）
```
选角色(v2) → 第1层(小盘,教学怪) → 三选一遗物 → 第2层(+商店格) 
→ 三选一 → 第3层(Boss怪:高强度+特殊规则) → 结算/无尽(v2)
```
- 每层清出楼梯即可下层（不必全清）——**全清有奖励**（宝箱），松紧自选。
- 层间**三选一遗物**（被动，改规则）；层内翻出**商店格**用金币买道具/回血。
- 升级：杀怪得 XP，升级回满血+最大血+1（Dragonsweeper 验证过的正反馈）。

### 内容表（v1 数值规模）
| 类别 | v1 数量 | 例子 |
|---|---|---|
| 怪物 | 8 种 | 史莱姆1 / 蝙蝠2（翻开时换位）/ 骷髅3 / 幽灵5（不贡献数字!）/ 宝箱怪（假金币）/ 龙13（Boss）|
| 主动道具 | 6 种 | 探针（免费翻一格）/ 扫描（显示3×3）/ 护盾（免下次伤害）/ 炸药（清3×3）/ 回血药 / 传送（换起点）|
| 遗物 | 9 种 | 「数字≤1 的格自动翻开」/「每翻 12 格回 1 血」/「怪物强度全体 -1」/「金币翻倍但商店涨价」（诅咒式）|
| 层 | 3 层 + Boss | 盘面 8×8 → 10×10 → 12×12 |

> 怪物设计原则照抄 Dragonsweeper 的精髓：**每种怪物 = 一条推理规则的扭曲**（幽灵不计入数字、蝙蝠会动……），图鉴逐步解锁，学习即进度。

### Meta 与留存（分期）
- v1：怪物图鉴收集 + 最佳成绩本地存档。
- v1.5：**Wordle 式 emoji 战报分享**（免费增长钩子）+ 种子每日盘（纯前端，无后端）。
- v2：遗物池解锁、多角色（多血型/道具型/透视型开局）、无尽模式。

### 变现钩子（engine Ads 现成）
- 激励视频：死亡复活 ×1 / 三选一刷新 / 全清宝箱双倍。
- 插屏：第 2 层结束起层间弹（沿用 2048 的节奏纪律）。

## 三、工程落地

- **引擎复用**：G + phase 状态机（HOME / RUN_MAP? 不要，直接 LEVEL_INTRO / PLAYING / PICK_RELIC / SHOP / WIN / LOSE）、renderAll/addHit、Input（本游戏无滑动，纯点击 + 长按）、i18n 十语、Ads/Portal、makeArt（emoji 占位 → ComfyUI 后补）。
- **长按插旗**：Input 需加长按支持（onLongPress）——引擎补一个回调，这是第一个由游戏反哺引擎的改动。
- **纯逻辑可单测**：盘面生成、数字计算、连锁展开、伤害结算、遗物效果全部写成无 DOM 纯函数 + node 单测（这正是选扫雷首发的原因）。
- **公平性保底**：起点保证 3×3 安全区；50/50 残局因血量制不再致命；真无猜求解器 = v2 的每日盘再说。
- 文案键结构：`mon.<id>.name/desc`、`item.*`、`relic.*`，en 基准，`node tools/check-locales.js games/minesweeper/locales`。

## 四、v1 范围（一周内可玩上线门户）

做：3 层 run、8 怪、6 道具、9 遗物、商店格、升级、图鉴、复活钩子、教学层、十语、emoji 占位美术。
不做：每日盘、排行榜、无尽、多角色、无猜求解器、真美术（占位先行）。
