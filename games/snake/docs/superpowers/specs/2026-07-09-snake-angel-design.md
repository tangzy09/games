# 天使贪吃蛇(Angel Snake)设计文档

日期:2026-07-09 ｜ 状态:已与用户逐节确认

## 1. 概述

可爱风贪吃蛇:蛇走过的格子永久揭开底下的「大头萌天使」图片,揭满全图过关换下一张。核心循环 = 揭图收集(500 张图鉴)+ 果子组合技 + 成就系统。带 AI 代打(数学上保证 100% 通关)、4 种可解锁皮肤、10 国语言、Google 广告变现。

- **素材源**:`Projects\language-study\images\` 共 3432 张统一画风 webp(512×512,~50KB/张),精选 500 张打包进本项目(自带副本,不跨项目引用)。
- **平台**:网页优先(EC2 `/var/www/games` 一仓多站、每游戏子域名,门户分发),数据验证后 Capacitor 上 store(monorepo README 的分发纪律)。
- **技术**:**接入 `Projects\games` monorepo 共享引擎**(`engine/`,从 2048 抽取,全局脚本+加载顺序,无 bundler)。游戏位于 `games/snake/`,实现引擎契约:单一状态对象 `G` + 全局 `renderAll()`(每帧 clearHits→全量重画→addHit)+ `dispatch(action)`;i18n/广告/门户/音频/平台探测/画布基建全部用引擎,**引擎只在 `engine/` 改,不许私拷**。

## 2. 核心玩法

- **棋盘**:16×16 = 256 格,正方形,天使图铺满底层。
- **揭图**:开局全棋盘被主题遮罩瓦片覆盖;蛇身经过的格子遮罩**永久消失**。相邻已揭格**无缝拼合**(整格方形挖洞+微重叠,不留缝不留角斑,天使图完整呈现——用户实测反馈)。顶部进度条显示 %。
- **过关**:256 格全揭 = 过关。全图展示动画(中心波纹亮出 + 彩带粒子),图片存入图鉴,给过关奖励分,随机换下一张未解锁图,遮罩重置,**蛇保留当前长度**继续。
- **单局定义**:一张图从遮罩重置到揭满的全过程(成就判定以此为界)。
- **死亡**(撞墙/撞自己):
  - 默认:蛇重生,长度 = **当前长度的一半**(最少 3),揭图进度保留,连击清零。(半长重生防「故意撞死当免费剪刀」)重生方式:仅蛇头置于安全格(距蛇尸最远的空旷区),身体随前进逐步「长出」至应有长度(同开局)。
  - 广告复活:看激励视频原地满状态复活(保长度、保连击),每局限 2 次。
- **速度**:基础 ~7 格/秒(待校准),随蛇长缓慢提升,有封顶。
- ~~按住加速~~:**已按用户实测反馈移除**(2026-07-10,P1 试玩后)。core.step 的 `freezeCombo` 选项保留为通用能力(有测试),P2 果子若需要可复用;UI 层不再有加速。
- **每关待机态(READY)**:开局与每次过关换图后,游戏处于待机——蛇不动,中央显示「准备好了吗?/开始!」;点按钮或按任意方向键开始(方向键会同时作为第一个转向输入);AI 代打开启时自动跳过待机。死亡重生(玩家主动点按钮)不进待机,直接继续。
- **操作**:手机全屏任意位置滑动转向;桌面方向键 / WASD。
- **连击**:吃到下一个果子间隔 ≤10 秒则连击 +1,超时不清零只停涨,**死亡才清零**。倍率随连击数提升(公式待校准)。
- **计分**(数值全部待校准,见 §13):果子基础分 × 连击倍率;揭图每 25% 里程碑奖励;过关大奖励。
- **收尾体验**:剩余未揭格 ≤10 时,未揭格遮罩发微光呼吸提示;同时提高揭图类果子权重。

## 3. 果子系统(13 种)

规则:场上**恒有 1 个普通苹果**;每吃 4~6 个苹果随机刷 1 个特殊果,**限时 8 秒**,消失前闪烁。

| 果子 | 效果 | 分类 |
|---|---|---|
| 🍎 苹果(普通) | +1 节、基础分 × 连击 | 基础 |
| 💫 双子星 | 场上额外多刷 2 个苹果 | 得分 |
| 👑 金苹果 | +50 分、连击 +2,**极稀有** | 得分 |
| 😈 小恶魔 | 提速 50% 持续 5 秒,期间得分 ×2 | 得分 |
| 🌠 流星 | **会移动**(斜穿棋盘一次),**飞过即沿途自动揭开轨迹**(不论是否追上);追上额外 +40 分,**稀有** | 得分 |
| 🌈 彩虹羽毛 | 随机揭开一片 3×3 未揭区域 | 揭图 |
| ✨ 圣光足迹 | 8 秒内蛇走过揭开 3 格宽光带 | 揭图 |
| ☁️ 慢慢云 | 减速 30%,持续 8 秒 | 生存 |
| ✂️ 天使之剪 | 蛇身缩短 3 节,**稀有** | 生存 |
| 😇 光环 | 6 秒幽灵模式,可穿过自己身体;**到期时若蛇头仍与身体重叠,宽限至脱离为止**(宽限期不可再穿入新的身体格) | 生存 |
| 💖 守护爱心 | 一层护盾:下次致死碰撞时**该步不执行,自动转向任一安全方向**,护盾消耗;若四方向均不安全(极罕见)仍判死 | 生存 |
| 🧲 磁力圣环 | 8 秒内场上果子向蛇头缓缓漂移(约 0.5s 一格,不漂入蛇身/出界/其他果子占用格) | 杂项 |
| 🎁 天国礼盒 | 随机触发其他任意一种果子的效果 | 杂项 |

**权重原则**:前期(蛇短)偏得分类;后期(蛇长或揭图 >60%)提高生存类与揭图类权重。权重表为配置项。

## 4. AI 算法(保证通关)

三层设计,AI 代打与「广告救场 10 秒」共用:

1. **哈密顿回路兜底**:16×16 偶数棋盘预计算 S 形遍历回路(格子编号 0~255)。严格沿回路走数学上永不撞自身,且每圈遍历全部格子 → 揭图 100% 必然完成。
2. **安全捷径**:每步检查邻格,允许沿回路编号「向前跳」抄近路吃果子,条件:目标编号在头尾之间的安全区间内且留余量;蛇长 >50% 棋盘后禁用捷径退回纯回路。捷径只允许穿越**已揭开**区域(未揭格留给回路自然扫过,保证不漏)。
3. **目标选择**:快消失特殊果 > 苹果 > 未揭区域;流星只顺路截击不强追。
   - **防揭图停滞**(关键,否则「保证通关」不成立——场上恒有苹果,未揭区域永远轮不上当目标):a) 寻路等价代价下**偏好穿过未揭格**;b) **停滞检测**:连续 N 步未新增揭格且无临期特殊果 → 强制以最近未揭格为目标;仍停滞 → 退回纯回路模式直至新增揭格(纯回路必扫全盘,保证性闭环)。
- **兜底的兜底**:若无安全走法(理论不发生),回退 BFS 追尾生存策略。
- **救场交还**:AI 救场 10 秒结束前 1 秒倒计时提示,交还瞬间不改变行进方向。
- **AI 代打的限制**(防掏空图鉴价值):照常解锁图鉴,但**得分减半、不判定单局成就、不计最高分、不更新历史纪录类累计成就**(历史最长蛇/历史最高连击);计数类累计成就(局数/苹果/揭图张数等)照常累加。
- 性能:每步 O(4),60fps 无压力。

## 5. 图鉴(500 张 / 25 集)

- `tools/pick-images.js` 从 language-study 抽 500 张拷入 `www/images/angels/`(~25MB),生成 `manifest.json`(500 id + 25 集 × 20 张分组 + 集名)。分组先随机,集名走可爱风,后期可人工调整。
- 图鉴 UI:25 集分页网格;未解锁 = 剪影 + 锁;已解锁可全屏欣赏、左右滑。
- **重温模式**:500 张集满后可随机重玩任意图,累计成就照常累加(长线成就的载体)。
- 加载:按需 lazy load 当前关图片,不预载全部。

- 适配器优先级:门户内 = Portal SDK 分成;自有站 = AdSense H5(`adBreak`,需 AFG 审核);App = AdMob(走 admob-monetization skill,含 **UMP GDPR 同意流程 + iOS ATT 提示**)。
- **无广告 fallback**:广告不可用时(审核期/无填充/断网),复活与 AI 救场降级为每日免费限次(复活 2 次/天,救场 3 次/天),功能不残废。
- **儿童受众政策风险**:可爱画风可能被商店判「面向儿童」(Play Families / Kids 类目),将强制儿童广告合规、eCPM 大跌。上架时受众定位选**全年龄休闲**、不进儿童类目、不在商店文案里主打儿童;若仍被强判,AdMob 切 `tagForChildDirectedTreatment` 合规模式。

## 11. 存档(引擎 `Platform` 存储门面,版本化)

键名一律 `CFG.key(...)` 前缀(`snake_*`),经引擎 `Platform` 读写(web=localStorage,Capacitor=Preferences,boot 时 `Platform.hydrate` 预载)。存档 JSON 结构:

```
{ v:1,
  settings:{ lang, theme, sound },
  gallery:{ unlocked:[id...], setsDone:[...] },
  ach:{ counters:{...}, unlocked:[...] },
  stats:{ 累计各项、历史纪录、连续天数、时长 },
  run:{ 当前局快照:蛇/方向/长度/揭图位图/当前图id/分数/连击 } }
```

过关/死亡/成就解锁/退出时写盘;启动检测 run 快照可续玩。

**iOS 防丢档**:WKWebView 的 localStorage 可能在存储压力下被系统清除。引擎 `Platform` 在 Capacitor 端以 **Preferences 插件为主存储**(原生持久层,不受 WKWebView 清理影响),网页端 localStorage——门面已隔离,游戏代码不感知。

## 12. 工程结构与测试

monorepo:`Projects\games\`(引擎与全部游戏一仓)。snake 结构:

```
games/                        ← monorepo 根
├─ engine/                    ← 共享引擎(config/platform/i18n/portal/ads/audio/canvas/input/controls)
│  └─ prng.js                 ← 本项目新增:可注种子 PRNG(全系列可用)
├─ tools/check-locales.js     ← 已有:locale 完整性校验
└─ games/snake/
   ├─ index.html              # engine 加载序 + GAME_CONFIG + 游戏脚本
   ├─ css/game.css            # 主题覆盖(engine.css 之上)
   ├─ js/ core.js ai.js fruits.js achievements.js render.js main.js
   │     (themes/gallery/share 等 P2 再加;i18n/ads/audio/storage 用引擎,不重写)
   ├─ locales/{en,zh-CN,...}.json
   ├─ assets/angels/ (500 webp + manifest.json)
   ├─ tools/pick-images.js
   ├─ tests/
   └─ docs/superpowers/
```

**引擎契约遵守**:`G` 单一状态对象 + `G.phase` 状态机;全局 `renderAll()` 每帧全量重画 + `addHit` 注册可点区域;所有交互过 `dispatch(action,data)`;boot 流程照引擎 README(Platform.hydrate → restoreAudioPrefs → Portal.boot → Ads.init → I18N → initCanvas → Input.bind → Controls.render → renderAll)。**引擎缺口在 engine/ 补**(P1:`prng.js` 新模块、`input.js` 加可选 `liveSwipe` 滑动即转向模式;P3:AdSense 适配)。

**工程约定**:
- 游戏内所有随机(果子刷新/选图/流星轨迹)统一走**可注种子 PRNG**(mulberry32),测试可复现、AI 断言确定性成立;
- `visibilitychange` 切后台**自动暂停**(手机来电/切 App 蛇不白死);
- canvas 分层:静态遮罩层 offscreen 缓存,只重绘脏格,粒子独立层。

**测试**(node 直跑,无框架):
- `test-ai.js`:**模拟 AI 连打 10 万步断言 0 死亡 + 每关必达 100%**(「保证通关」的机器验证);
- `test-achievements.js`:120 成就计数器逐个触发断言;
- `test-fruits.js`:权重表合法性、效果状态机;
- `test-storage.js`:存档版本迁移与快照恢复。
